Avaliação de Alertas

21.0 Introdução

Uma habilidade crítica para um analista de segurança cibernética é a capacidade de avaliar alertas e decidir o que fazer com eles. Neste módulo, você aprenderá de onde vêm os alertas, fluxos de trabalho comuns associados a alertas e métodos padrão para avaliar e classificar alertas.

Objetivo do Módulo: Explicar o processo de avaliação de alertas

21.1 Fontes de Alertas

21.1.1 Security Onion

Security Onion é um pacote de código aberto de ferramentas de Monitoramento de Segurança de Rede (NSM) que é executado em uma distribuição Ubuntu Linux. As ferramentas Security Onion fornecem três funções principais para o analista de segurança cibernética:

  • Captura completa de pacotes e tipos de dados
  • Sistemas de detecção de intrusão baseados em rede e em host
  • Ferramentas de analistas de alerta
 Observação: Security Onion pode ser abreviado como SO. Neste curso, usaremos Security Onion.

CapMe

Aplicação web que permite visualização de transcrições pcap renderizadas com tcpflow ou Zeek. Oferece ao analista um meio fácil de visualizar uma sessão inteira da Camada 4.

Snort

NIDS (Network Intrusion Detection System) que usa regras e assinaturas para gerar alertas. Patrocinado pela Cisco, pode baixar novas regras automaticamente via PulledPork.

Zeek (Bro)

NIDS baseado em comportamento. Usa políticas em forma de scripts para determinar quais dados registrar e quando emitir notificações de alerta.

OSSEC

HIDS (Host Intrusion Detection System) que monitora integridade de arquivos, logs locais, processos do sistema e detecção de rootkit.

Wazuh

HIDS que substituirá OSSEC. Solução completa com análise de logs, monitoramento de integridade, detecção de vulnerabilidades e resposta a incidentes.

Suricata

NIDS com abordagem baseada em assinaturas. Usa multithreading nativo e suporte a GPU para melhor desempenho.

21.1.3 Ferramentas de Análise

O Security Onion integra dados e logs de IDS através das seguintes ferramentas:

📊 Sguil

Console de alto nível para investigar alertas de segurança. Serve como ponto de partida na investigação, permitindo girar diretamente para outras ferramentas.

📈 Kibana

Interface de painel interativo para dados Elasticsearch. Permite consulta de dados e visualizações flexíveis com recursos de machine learning.

🔍 Wireshark

Aplicativo de captura de pacotes integrado que pode ser aberto diretamente de outras ferramentas para exibir pacotes completos relevantes.

📡 Zeek

Analisador de tráfego de rede que inspeciona todo o tráfego em um segmento, permitindo análise aprofundada dos dados.

21.1.4 Geração de Alertas

Alertas de segurança são mensagens de notificação geradas por ferramentas, sistemas e dispositivos de segurança do NSM. No Security Onion, o Sguil fornece um console que integra alertas de várias fontes em uma fila com carimbo de data/hora.

Five-Tuple (5-tupla) em Alertas: SRCIP, SPORT, DSTIP, DPORT, Protocolo

Interface Sguil - Eventos em Tempo Real

ST CNT Sensor Alert ID Date/Time Src IP SPort Dst IP DPort Pr Event Message

ST: Status (RT = Tempo Real) - Cores indicam prioridade
CNT: Contagem de eventos correlacionados para mesmo IP
Alert ID: Sensor ID + Número do evento
Prioridades: Muito baixo (amarelo claro) → Alto (vermelho)

21.1.5-6 Regras e Alertas Snort

alert tcp $EXTERNAL_NET any -> $HOME_NET 21 (msg: "ET EXPLOIT VSFTPD Backdoor User Login Smiley"; flow:established,to_server; content: "USER "; depth:5; content:"|3a 29|"; distance:0; classtype:attempted-admin; sid:2013188; rev:4;)
/nsm/server_data/securityonion/rules/seconion-eth1-1/download.rules: Line 7159

Estrutura da Regra Snort

Componente Exemplo Explicação
Cabeçalho da Regra alert ip any any -> any any Ação, protocolo, endereços/portas de origem/destino, direção
Opções de Regra (msg:"GPL ATTACK_RESPONSE id check returned root";...) Mensagem, conteúdo do pacote, tipo de alerta, SID
Local da Regra /nsm/server_data/securityonion/rules/... Caminho do arquivo e linha da regra

Cabeçalho da Regra - Componentes

Componente Explicação
alert Ação: emitir um alerta
ip Protocolo
any any Origem: qualquer IP e porta
-> Direção do fluxo (origem → destino)
any any Destino: qualquer IP e porta

Opções de Regra - Componentes

Opção Explicação
msg: Texto descritivo do alerta
content: Conteúdo do pacote a ser inspecionado
reference: URL com mais informações sobre a regra
classtype: Categoria do ataque (prioridade associada)
sid: Identificador único da regra
rev: Revisão da regra
Fontes de Regras Snort:
  • GPL - Regras antigas Sourcefire (Snort SIDs ≤ 3464)
  • ET - Emerging Threats (código aberto, licença BSD)
  • VRT - Cisco Talos (assinantes, liberadas após 30 dias)
