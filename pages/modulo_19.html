Módulo 19: Tecnologias e Protocolos

Explicar como as tecnologias de segurança afetam o monitoramento de segurança.

Monitoramento de Protocolos Tecnologias de Segurança Syslog & NTP DNS Exfiltração

19.0 Introdução

Por que devo usar este módulo?

Os bons protocolos podem ter maus usos. Às vezes, os protocolos que usamos diariamente podem ser virados contra nós. Existem muitos protocolos diferentes que são usados nas operações diárias de uma rede. Este módulo explicará como esses protocolos interagem e impactam o monitoramento da segurança da rede.

Objetivos do Módulo:
  • ✓ Explicar o comportamento dos protocolos de rede comuns no contexto do monitoramento de segurança
  • ✓ Explicar como as tecnologias de segurança afetam a capacidade de monitorar protocolos de rede comuns

19.1 Monitorando Protocolos Comuns

19.1.1 Syslog e NTP

Vários protocolos que geralmente aparecem em redes têm recursos que os tornam de especial interesse no monitoramento de segurança. Por exemplo, syslog e Network Time Protocol (NTP) são essenciais para o trabalho do analista de segurança cibernética.

O padrão syslog é usado para registrar mensagens de eventos de dispositivos de rede e endpoints. O padrão permite um meio neutro de sistema de transmissão, armazenamento e análise de mensagens. Muitos tipos de dispositivos de vários fornecedores diferentes podem usar syslog para enviar entradas de log para servidores centrais que executam um daemon syslog. 

# Syslog - Porta UDP 514
# Centralização de logs para monitoramento de segurança
Dispositivos de Rede
Servidor Syslog
Estação de Monitoramento

Fluxo de logs centralizados

Ameaças ao Syslog: Atores maliciosos podem atacar servidores syslog para ocultar evidências de explorações, bloquear transferência de dados ou destruir logs. Syslog-ng oferece aprimoramentos de segurança.
Pesquise: syslog-ng

19.1.2 NTP (Network Time Protocol)

As mensagens do Syslog geralmente são carimbadas de data e hora. Isso permite que mensagens de diferentes fontes sejam organizadas pelo tempo para fornecer uma visão dos processos de comunicação de rede. Como as mensagens podem vir de muitos dispositivos, é importante que os dispositivos compartilhem um timeclock consistente.

O Network Time Protocol (NTP) usa uma hierarquia de fontes de tempo autoritativas para compartilhar informações de tempo entre dispositivos na rede. O NTP opera na porta UDP 123. 

# NTP - Porta UDP 123
# Sincronização de tempo para correlação de eventos

Fonte Autoritativa → Servidor NTP Local → Dispositivos

Ameaças NTP: Atores podem atacar infraestrutura NTP para corromper informações de tempo (ofuscando rastros) ou usar servidores NTP em ataques DDoS por amplificação.

19.1.3 DNS - Exfiltração de Dados

O Serviço de Nome de Domínio (DNS) é usado por milhões de pessoas diariamente. Por isso, muitas organizações têm políticas menos rigorosas para proteger contra ameaças baseadas em DNS. Os invasores reconheceram isso e geralmente encapsulam diferentes protocolos de rede no DNS para evitar dispositivos de segurança.

Algumas variedades de malware usam DNS para se comunicar com servidores de comando e controle (CNC) e para exfiltrar dados no tráfego disfarçados como consultas DNS normais. Vários tipos de codificação, como Base64, binário de 8 bits e Hex podem ser usados para camuflar os dados. 

# Exemplo de exfiltração via DNS
aW4gcGxhY2UgDg8gcHJvdGVjdC.example.com
BhZ2FpbnN0IEROUyBiYXNIZCBOa.example.com
HJJYXRzIHRoYW4gdGhleSBoyXZI.example.com
# Dados codificados em Base64 como subdomínios

Dados Exfiltrados → Subdomínio DNS → Servidor DNS Comprometido

Base64 Hex 8-bit Binary
Detecção: Subdomínios extremamente longos ou com aparência aleatória são indicadores de exfiltração DNS. Analise logs de proxy DNS ou use serviços como Cisco Umbrella para bloquear domínios suspeitos.

19.1.4 HTTP e HTTPS

O Hypertext Transfer Protocol (HTTP) é o protocolo de backbone da World Wide Web. No entanto, todas as informações transportadas em HTTP são transmitidas em texto simples do computador de origem para o destino na internet. O HTTP não protege os dados contra alteração ou interceptação.

Injeção iFrame

Uma exploração comum de HTTP é chamada de injeção iFrame (quadro inline). Um ator de ameaça compromete um servidor da Web e planta código malicioso que cria um iFrame invisível em uma página da Web comumente visitada. Quando o iFrame é carregado, o malware é baixado.

Site Confiável → iFrame Malicioso → Download de Malware

A filtragem Cisco Web Reputation detecta conteúdo de sites não confiáveis

HTTPS e Criptografia

HTTPS adiciona uma camada de criptografia ao protocolo HTTP usando SSL/TLS. Isso torna os dados HTTP ilegíveis em trânsito. No entanto, o tráfego HTTPS criptografado complica o monitoramento de segurança de rede. 

Cliente → [SSL/TLS] → Servidor
       Chave Simétrica
       Certificado Digital
Transação HTTPS: Cliente solicita página segura → Servidor envia certificado → Cliente valida → Cria chave simétrica → Comunicação criptografada.

19.1.5 Protocolos de Email (SMTP, POP3, IMAP)

Protocolos de e-mail como SMTP, POP3 e IMAP podem ser usados por atores de ameaças para espalhar malware, exfiltrar dados ou fornecer canais para servidores CNC de malware.

  • SMTP (porta 25): Envia dados de um host para servidor de email. Usado para exfiltração de dados (ex: Hack da Sony Pictures 2014).
  • POP3/IMAP: Usados para baixar mensagens. Responsáveis por trazer malware para o host.
Host Infectado
SMTP → Extração de Dados
Servidores CNC

19.1.6 ICMP - Túnel e Exfiltração

ICMP tem muitos usos legítimos, no entanto, a funcionalidade ICMP também tem sido usada para criar vários tipos de explorações. O ICMP pode ser usado para identificar hosts em uma rede, a estrutura de uma rede e determinar os sistemas operacionais em uso. 

# Ameaças ICMP
- Identificação de hosts
- Mapeamento de rede
- Fingerprint de SO
- Ataques DoS
- Túnel ICMP (exfiltração de dados)
Exploração LOKI: Conhecida exploração que usa túnel ICMP para transferir arquivos de hosts infectados.
Pesquise: Ping Tunnel, LOKI exploit

19.2 Tecnologias de Segurança

19.2.1 ACLs - Mitigando Abuso ICMP

Listas de Controle de Acesso (ACLs) podem dar uma falsa sensação de segurança se forem excessivamente confiadas. As ACLs e a filtragem de pacotes em geral são tecnologias que contribuem para um conjunto em evolução de proteções de segurança de rede. 

# ACL para tráfego ICMP da Internet
access-list 112 permit icmp any any echo-reply
access-list 112 permit icmp any any source-quench
access-list 112 permit icmp any any unreachable
access-list 112 deny icmp any any
access-list 112 permit ip any any
# ACL para tráfego ICMP interno
access-list 114 permit icmp 192.168.1.0 0.0.0.255 any echo
access-list 114 permit icmp 192.168.1.0 0.0.0.255 any parameter-problem
access-list 114 permit icmp 192.168.1.0 0.0.0.255 any packet-too-big
access-list 114 permit icmp 192.168.1.0 0.0.0.255 any source-quench
access-list 114 deny icmp any any
access-list 114 permit ip any any
Limitações das ACLs: Invasores podem determinar quais IPs, protocolos e portas são permitidos via varredura. Podem criar pacotes com endereços falsificados. Firewalls de próxima geração (NGFW), AMP e appliances de segurança são necessários para defesa avançada.

19.2.2 NAT e PAT

Tradução do Endereço da Rede (NAT) e Tradução de Endereço de Porta (PAT) podem complicar o monitoramento de segurança. Vários endereços IP são mapeados para um ou mais endereços públicos visíveis na Internet, ocultando os endereços IP individuais que estão dentro da rede.

Rede Interna

PC1: 192.168.10.10 (Endereço Interno)

NAT

Rede Externa

SA: 209.165.200.226 (Endereço Global)

DA: 209.165.201.1 (Servidor Web)

Impacto no Monitoramento:
  • Dificulta identificar dispositivo interno específico
  • Fluxos NetFlow são "quebrados" pelo NAT
  • Cisco oferece produtos que "costuram" fluxos mesmo após NAT

19.2.3 Criptografia, Encapsulamento e Tunelamento

A criptografia pode apresentar desafios para o monitoramento de segurança tornando os detalhes do pacote ilegíveis. A criptografia faz parte das tecnologias VPN. Nas VPNs, um protocolo comum, como IP, é usado para transportar tráfego criptografado.

Túnel Criptografado

Host Interno ↔ Túnel ↔ Dispositivos de Ameaças

Ameaça: Malware pode estabelecer túnel criptografado usando protocolos comuns e confiáveis para exfiltrar dados da rede.

19.2.4 Rede Ponto a Ponto (P2P) e Tor

P2P

Na rede ponto a ponto (P2P), os hosts podem operar em funções de cliente e servidor. Três tipos principais:

  • Compartilhamento de arquivos (Napster, Gnutella, BitTorrent)
  • Compartilhamento de processadores (pesquisa científica)
  • Mensagens instantâneas (Webex Teams)
Riscos P2P: Contorna firewalls, vetor comum de malware, arquivos frequentemente infectados, portas dinâmicas.

Tor (The Onion Router)

Tor permite navegação anônima através de uma rede de relés. O tráfego é criptografado em camadas (como uma cebola), removidas a cada salto.

Usuário → Relé Tor → Relé Tor → Relé Tor → Destino
(Criptografia em camadas)
Desafios: Usado por criminosos na Dark Net, canal de comunicação CNC, ofusca endereços IP de destino.

19.2.5 Balanceamento de Carga com Delegação DNS

O balanceamento de carga envolve a distribuição do tráfego entre dispositivos ou caminhos de rede para evitar recursos sobrecarregados. Uma maneira de fazer isso na internet é através de técnicas que usam DNS para enviar tráfego para recursos que têm o mesmo nome de domínio, mas vários endereços IP.

1
Usuário deseja visitar www.example.com. Consulta DNS enviada.
2
Servidor DNS local não possui registro, consulta outros servidores.
3
Solicitação chega ao servidor DNS autoritativo. Delegados do NS solicitam balanceador de carga.
4
Balanceador de carga retorna IP do servidor no pool dependendo da carga.

Load Balancer (LBM)

Testes de integridade e desempenho de caminhos podem parecer tráfego suspeito para analistas não familiarizados.

19.2.6 Verifique sua compreensão – Identifique o impacto da tecnologia na segurança e no monitoramento

Responda às perguntas para identificar o impacto da tecnologia na segurança da rede e no monitoramento da segurança.

1

Qual das seguintes opções espalha arquivos infectados por malware e cria vulnerabilidades em uma rede?

A ACL
B criptografia
C Tunelamento
D NAT/PAT
E P2P
Resposta Correta: P2P
Redes ponto a ponto (P2P) são vetores comuns para propagação de malware. Arquivos compartilhados frequentemente contêm malware, e atores de ameaças podem posicionar seu malware em clientes P2P para distribuição a outros usuários. Além disso, o P2P pode contornar firewalls e usar numeração dinâmica de portas, criando vulnerabilidades adicionais.
2

Qual das seguintes opções pode ser derrotado por pacotes contendo endereços IP falsificados?

A ACL
B criptografia
C TOR
D NAT/PAT
E P2P
Resposta Correta: ACL
Listas de Controle de Acesso (ACLs) podem ser derrotadas por pacotes contendo endereços IP falsificados (spoofing). Invasores podem criar pacotes que usam endereços IP de origem falsificados para contornar as regras configuradas. Firewalls de próxima geração (NGFW) e medidas baseadas em contexto são necessárias para mitigar essa vulnerabilidade.
3

Qual das seguintes opções torna o conteúdo da mensagem e anexos de arquivos ilegíveis?

A ACL
B criptografia
C TOR
D NAT/PAT
E Balanceamento de carga
Resposta Correta: Criptografia
A criptografia torna o conteúdo das mensagens e anexos de arquivos ilegíveis para qualquer pessoa que não possua a chave de descriptografia adequada. Isso inclui tecnologias como SSL/TLS (HTTPS), VPNs e criptografia de email. Embora essencial para privacidade, a criptografia também complica o monitoramento de segurança, pois oculta o conteúdo do tráfego.
4

Qual das seguintes opções oculta os detalhes de identidade dos usuários que estão navegando na Web?

A ACL
B P2P
C TOR
D NAT/PAT
E Balanceamento de carga
Resposta Correta: TOR (The Onion Router)
O TOR é uma plataforma que permite navegação anônima na internet. Ele constrói um caminho criptografado em camadas através de uma rede de relés, onde cada camada é "removida" a cada salto. Nenhum dispositivo único conhece todo o caminho para o destino, e o endereço IP de origem é ocultado, proporcionando anonimato ao usuário.

19.3 Resumo de Tecnologias e Protocolos

Consolidação dos principais conceitos abordados neste módulo sobre monitoramento de protocolos e tecnologias de segurança.

Monitorando Protocolos Comuns

Syslog

  • Porta UDP 514
  • Centraliza logs de dispositivos de rede
  • Alvo de ataques para ocultar evidências
  • Syslog-ng oferece aprimoramentos de segurança

NTP (Network Time Protocol)

  • Porta UDP 123
  • Sincroniza tempo entre dispositivos
  • Essencial para correlacionar eventos de segurança
  • Pode ser alvo de ataques DDoS por amplificação

DNS (Domain Name System)

  • Usado para exfiltração de dados via subdomínios
  • Codificação Base64, Hex e binária para camuflar dados
  • Comunicação com servidores CNC
  • Subdomínios longos indicam atividade suspeita

HTTP e HTTPS

  • HTTP: Texto plano, vulnerável a interceptação
  • Injeção iFrame: Malware escondido em páginas confiáveis
  • HTTPS: Criptografia SSL/TLS, mas complica monitoramento
  • Inspeção SSL apresenta desafios de processamento e privacidade

Protocolos de Email

  • SMTP: Exfiltração de dados (ex: Hack Sony 2014)
  • POP3/IMAP: Vetores de entrada de malware
  • Tráfego SMTP frequentemente não monitorado
  • Análise retrospectiva rastreia comportamento de malware

ICMP

  • Identifica hosts, mapeia rede, fingerprint de SO
  • Usado em ataques DoS
  • Túnel ICMP: Exfiltração de dados (exploração LOKI)
  • Tráfego ICMP frequentemente ignorado, criando brecha de segurança

Tecnologias de Segurança

ACLs (Access Control Lists)

  • Podem dar falsa sensação de segurança
  • Vulneráveis a IP spoofing e varredura de portas
  • Não antecipam técnicas emergentes de manipulação
  • Firewalls NGFW oferecem proteção avançada

NAT e PAT

  • Ocultam endereços IP internos
  • Dificultam identificação de dispositivos específicos
  • Quebram fluxos NetFlow (unidirecionais)
  • Cisco oferece soluções para "costurar" fluxos após NAT

Criptografia e Tunelamento

  • Torna conteúdo de pacotes ilegível
  • VPNs criam conexões virtuais ponto a ponto
  • Malware pode criar túneis criptografados para exfiltração
  • Desafia monitoramento de segurança tradicional

P2P (Peer-to-Peer)

  • Compartilhamento de arquivos, processadores e IM
  • Contorna firewalls, usa portas dinâmicas
  • Vetor comum de propagação de malware
  • Não recomendado em redes corporativas

TOR (The Onion Router)

  • Roteamento em camadas (onion routing)
  • Navegação anônima na internet
  • Usado por organizações criminosas na Dark Net
  • Canal de comunicação para malware CNC
  • Ofusca endereços IP, evita listas negras

Balanceamento de Carga

  • Distribui tráfego entre múltiplos recursos
  • Delegação DNS com múltiplos endereços IP
  • Testes de integridade podem parecer tráfego suspeito
  • Múltiplos IPs podem aparecer como anomalias em capturas

19.3.1 O que eu aprendi neste módulo?

Monitorando protocolos comuns:

Compreendi como protocolos como Syslog, NTP, DNS, HTTP/HTTPS, email e ICMP podem ser tanto ferramentas essenciais quanto vetores de ataque. Aprendi a identificar sinais de exfiltração DNS, túneis ICMP, injeção iFrame e os desafios que a criptografia HTTPS impõe ao monitoramento.

Limitações das ACLs:

Entendi que ACLs podem ser contornadas por IP spoofing, varredura de portas e técnicas de manipulação de pacotes, exigindo soluções mais avançadas como firewalls de próxima geração.

Impacto de NAT/PAT no monitoramento:

Aprendi como a tradução de endereços oculta IPs internos e quebra fluxos NetFlow, complicando a correlação de eventos de segurança.

Criptografia e tunelamento:

Compreendi que, embora essencial para privacidade, a criptografia cria desafios significativos para monitoramento, podendo ser explorada por malware para criar túneis de exfiltração.

Riscos de P2P e TOR:

Identifiquei como redes P2P propagam malware e como o TOR oferece anonimato que pode ser explorado por atividades criminosas e comunicação CNC.

Balanceamento de carga DNS:

Aprendi que técnicas de balanceamento podem gerar tráfego que parece suspeito, exigindo que analistas reconheçam esses padrões legítimos.

🌍 Casos Reais: Quando isso aconteceu no mundo real

🔴 Exfiltração DNS

2020 · Vasta Plataforma

O que aconteceu: Hackers invadiram a Vasta, empresa que gerencia escolas, e roubaram dados de 500 mil alunos e professores. Eles usaram exfiltração DNS para enviar os dados roubados para fora da rede sem serem detectados.

Analogia: Imagine um ladrão que esconde itens roubados dentro de caixas que parecem entregas normais. Ninguém desconfia porque as caixas são enviadas o tempo todo.
Lição: Monitorar comprimento de subdomínios DNS pode revelar exfiltração. Subdomínios muito longos são suspeitos!

🎬 Sony Pictures Hack

2014 · Sony Pictures

O que aconteceu: Hackers invadiram a Sony Pictures, roubaram dados confidenciais e usaram SMTP (email) para exfiltrar informações. Eles enviaram dados roubados disfarçados de emails normais.

Analogia: É como um funcionário que coloca documentos secretos dentro de envelopes de correspondência comum e os envia pelo correio da empresa.
Lição: Tráfego SMTP precisa ser monitorado mesmo sendo comum. O volume não deve ser desculpa para ignorar!

⏰ Ataque NTP Amplification

2014 · Cloudflare (1.7 Tbps)

O que aconteceu: Hackers usaram servidores NTP mal configurados para lançar um dos maiores ataques DDoS da história, atingindo 1.7 Tbps contra a Cloudflare.

Analogia: Imagine ligar para um serviço de informação e pedir para eles ligarem de volta para um número com uma mensagem enorme. Se muitos fizerem isso, o alvo recebe milhares de ligações ao mesmo tempo.
Lição: Servidores NTP devem ser configurados corretamente e não expostos publicamente sem restrições.

📰 Injeção iFrame em portais brasileiros

2018 · Diversos sites .gov.br

O que aconteceu: Sites governamentais brasileiros foram comprometidos com código iFrame malicioso que redirecionava visitantes para páginas de phishing e malware.

Analogia: Como colocar um cartaz falso em um quadro de avisos oficial. Quem vê no mural da prefeitura acredita que é informação legítima.
Lição: Sites confiáveis também podem ser comprometidos. Filtros de reputação web ajudam a bloquear conteúdo malicioso.

🐟 Exploração LOKI (Túnel ICMP)

1996 · Criada por daemon9

O que aconteceu: A ferramenta LOKI demonstrou como é possível enviar comandos e dados através de pacotes ICMP (ping), contornando firewalls que normalmente permitem esse tráfego.

Analogia: É como usar um interfone de prédio (que todo mundo usa para falar com portaria) para enviar mensagens secretas, e ninguém desconfia porque parece uma conversa normal.
Lição: Nunca confie cegamente em protocolos "inofensivos". O ICMP precisa ser monitorado como qualquer outro tráfego.

🎵 P2P e propagação de malware

2000-2010 · Kazaa, LimeWire, eMule

O que aconteceu: Redes P2P como Kazaa e LimeWire se tornaram os maiores vetores de propagação de malware da época. Arquivos de música e filmes populares continham código malicioso disfarçado.

Analogia: Imagine uma feira livre onde qualquer pessoa pode montar uma barraca. Criminosos colocam produtos com preços baixos, mas dentro deles há algo perigoso escondido.
Lição: Em redes corporativas, aplicativos P2P devem ser bloqueados. Em casa, baixe apenas de fontes oficiais.
