20.1 Tipos de Dados de Segurança

20.1.1 Dados de Alerta

Os dados de alerta consistem em mensagens geradas por sistemas de prevenção de intrusões (IPSs) ou sistemas de detecção de intrusões (IDSs) em resposta ao tráfego que viola uma regra ou corresponde à assinatura de uma exploração conhecida.

Um IDS de rede (NIDS), como o Snort, vem configurado com regras para explorações conhecidas. Os alertas são gerados pelo Snort e são legíveis e pesquisáveis pelos aplicativos Sguil e Squert, que fazem parte do conjunto Security Onion de ferramentas NSM. 

# Regra Snort para detecção
alert ip any any -> any any (msg:"GPL ATTACK_RESPONSE id check returned root"; 
content:"uid=0|28|root|25"; sid:2100498; rev:8;)

Site de teste: testmyds.com

Retorna o texto: uid=0(root) gid=0(root) groups=0(root)

Snort Sguil Security Onion
Importante: O site testmyds é uma ferramenta inofensiva usada para verificar se o NIDS está funcionando corretamente. Quando um host visita este site, uma assinatura é correspondida e um alerta é acionado.

20.1.2 Dados de Sessão

Os dados de sessão são um registro de uma conversa entre dois pontos de extremidade de rede, que geralmente são um cliente e um servidor. Dados de sessão são dados sobre a sessão, não os dados recuperados e usados pelo cliente.

Campos do Dado de Sessão (Zeek)

1. ts: carimbo de data e hora
2. uid: ID de sessão única
3. id_orig_h: IP de origem
4. id_orig_p: porta de origem
5. id_resp_h: IP de destino
6. id_resp_p: porta de destino
7. proto: protocolo de transporte
8. service: protocolo de aplicação
9. duration: duração da sessão
10. orig_bytes: bytes do host de origem
11. resp_bytes: bytes do host respondente
12. orig_packets: pacotes do host de origem
13. resp_packets: pacotes do host respondente

As Cinco Tuplas

  • IP Origem
  • Porta Origem
  • IP Destino
  • Porta Destino
  • Protocolo

Zeek (anteriormente Bro) é uma ferramenta de monitoramento de segurança de rede.

20.1.3 Dados de Transação

Os dados de transação consistem nas mensagens que são trocadas durante sessões de rede. Essas transações podem ser exibidas em transcrições de captura de pacotes. Os logs de dispositivos mantidos por servidores também contêm informações sobre as transações que ocorrem entre clientes e servidores. 

# Requisição HTTP
GET /home/index.html HTTP/1.1
Host: www.example.com
User-Agent: Mozilla/5.0 (Windows NT 6.1; WOW64; rv:53.0)

# Resposta HTTP
HTTP/1.1 200 OK
Date: Fri, 10 Oct 2015 23:59:59 GMT
Content-Type: text/plain

<text returned>
# Log de Acesso
192.168.1.10 - anyUser [10/Oct/2015:13:55:36 -0500] 
"GET /index.html HTTP/1.1" 200 326
Diferença: A sessão é todo o tráfego envolvido na elaboração da solicitação; a transação é a própria solicitação.

20.1.4 Capturas Completas de Pacotes

Capturas completas de pacotes são os dados de rede mais detalhados que geralmente são coletados. Devido à quantidade de detalhes, eles também são os tipos de dados mais intensos de armazenamento e recuperação.

As capturas completas de pacotes contêm não apenas dados sobre conversas de rede, como dados de sessão, mas também o conteúdo real das conversas:

  • Texto das mensagens de email
  • HTML nas páginas da Web
  • Arquivos que entram ou saem da rede

A ferramenta familiar Wireshark é muito popular para visualizar capturas de pacotes completos.

20.1.5 Dados Estatísticos

Os dados estatísticos são criados através da análise de outras formas de dados de rede. Podem ser feitas conclusões que descrevem ou predizem o comportamento da rede a partir dessas análises.

NBA / NBAD

Network Behavior Analysis e Network Behavior Anomaly Detection usam técnicas analíticas avançadas para analisar dados NetFlow ou IPFIX.

Análise Preditiva IA / Machine Learning Detecção de Anomalias
Cisco Cognitive Threat Analytics

Produto baseado em nuvem que usa aprendizado de máquina e modelagem estatística para:

  • Criar linha de base do tráfego
  • Identificar anomalias
  • Analisar comportamento de usuários e dispositivos
  • Detectar comunicação CNC e exfiltração

20.2 Logs de Dispositivos Finais

Os dispositivos finais (endpoints) geram logs que são essenciais para o monitoramento de segurança. Estes incluem:

  • Logs de Sistema: Eventos do sistema operacional
  • Logs de Aplicações: Atividades de software instalado
  • Logs de Segurança: Tentativas de login, alterações de privilégios
  • Logs de Autenticação: Sucessos e falhas de autenticação
Importante: A correlação entre logs de endpoints e logs de rede permite identificar a origem e o impacto de incidentes de segurança. 
# Exemplo de Log de Autenticação (Linux)
Mar 31 15:30:22 servidor sshd[12345]: Failed password for user admin from 192.168.1.100 port 54321 ssh2
Mar 31 15:30:25 servidor sshd[12345]: Accepted password for user admin from 192.168.1.100 port 54322 ssh2

# Exemplo de Log de Sistema (Windows)
Event ID 4624: Account Successfully Logged On
Event ID 4625: Account Failed to Log On

20.3 Logs de Rede

Os logs de rede são coletados de diversos dispositivos de infraestrutura:

  • Roteadores e Switches: Logs de interface, erros de roteamento
  • Firewalls: Conexões permitidas/negadas, NAT
  • Servidores DNS: Consultas de resolução de nomes
  • Servidores DHCP: Atribuição de endereços IP
  • NetFlow/IPFIX: Dados de fluxo de rede
Ferramentas comuns: Syslog-ng, ELK Stack (Elasticsearch, Logstash, Kibana), Splunk, Security Onion

20.2.5 Verifique sua compreensão - Identifique os eventos de segurança do Windows

Verifique sua compreensão do registro em log do Windows identificando o nível de gravidade para eventos específicos.

1

Qual nível de segurança do Windows é registrado quando um logon remoto foi tentado sem êxito por um usuário não autorizado?

A Erro
B Fatal
C Aviso
D Informações
E Sucesso na Auditoria
F Falha na Auditoria
Resposta Correta: Falha na Auditoria
Eventos de "Falha na Auditoria" são registrados quando uma tentativa de logon ou acesso falha, especialmente quando um usuário não autorizado tenta acessar o sistema. Isso é crucial para detectar tentativas de invasão e ataques de força bruta. O Event ID 4625 no Windows registra falhas de logon.
2

Qual nível de segurança do Windows é registrado quando um processo necessário foi carregado com êxito em uma estação de trabalho?

A Erro
B Fatal
C Aviso
D Informações
E Sucesso na Auditoria
F Falha na Auditoria
Resposta Correta: Informações
O nível "Informações" é usado para registrar eventos normais e bem-sucedidos do sistema, como a inicialização de serviços, carregamento de processos e outras operações que ocorrem conforme o esperado. Esses eventos ajudam a estabelecer uma linha de base do comportamento normal do sistema.
3

Qual nível de segurança do Windows é registrado quando ocorreu um erro de disco durante uma operação de backup?

A Erro
B Fatal
C Aviso
D Informações
E Sucesso na Auditoria
F Falha na Auditoria
Resposta Correta: Erro
O nível "Erro" é usado para registrar problemas significativos que afetam a funcionalidade do sistema, como falhas de hardware, erros de disco ou serviços que não conseguem iniciar. Esses eventos requerem atenção do administrador, pois podem indicar problemas que precisam ser resolvidos.
4

Qual nível de segurança do Windows é registrado quando o espaço em disco está ficando baixo?

A Erro
B Fatal
C Aviso
D Informações
E Sucesso na Auditoria
F Falha na Auditoria
Resposta Correta: Aviso
O nível "Aviso" é usado para registrar condições que não são imediatamente críticas, mas podem se tornar problemas se não forem tratadas. Espaço em disco baixo, falhas de driver recuperáveis ou configurações de segurança não ideais são exemplos típicos de eventos de aviso.
5

Qual nível de segurança do Windows é registrado quando um usuário autorizado faz logon remotamente em um sistema?

A Erro
B Fatal
C Aviso
D Informações
E Sucesso na Auditoria
F Falha na Auditoria
Resposta Correta: Sucesso na Auditoria
O nível "Sucesso na Auditoria" registra eventos de segurança bem-sucedidos, como logons autorizados, alterações de privilégios e acessos a recursos. Isso é essencial para rastrear atividades legítimas e estabelecer uma trilha de auditoria. O Event ID 4624 no Windows registra logons bem-sucedidos.

20.3 Logs de Rede

20.3.1 Tcpdump

A ferramenta de linha de comando tcpdump é um analisador de pacotes muito popular. Ele pode exibir capturas de pacotes em tempo real ou gravar capturas de pacotes em um arquivo. Ele captura dados detalhados de protocolo de pacotes e conteúdo.

Wireshark é uma GUI construída sobre a funcionalidade tcpdump. A estrutura das capturas tcpdump varia dependendo do protocolo capturado e dos campos solicitados. 

# Comandos básicos do tcpdump
tcpdump -i eth0                    # Capturar na interface eth0
tcpdump -w captura.pcap            # Salvar captura em arquivo
tcpdump -r captura.pcap            # Ler arquivo de captura
tcpdump port 80                    # Filtrar por porta
tcpdump host 192.168.1.1           # Filtrar por endereço IP
tcpdump -n                         # Não resolver nomes DNS

tcpdump → Wireshark

Análise de pacotes em tempo real e offline

CLI GUI PCAP

20.3.2 NetFlow

NetFlow é um protocolo desenvolvido pela Cisco como uma ferramenta para solução de problemas de rede e contabilidade baseada em sessão. O NetFlow fornece com eficiência um importante conjunto de serviços para aplicativos IP, incluindo:

  • Contabilidade de tráfego de rede
  • Faturamento de rede com base no uso
  • Planejamento de rede
  • Segurança e monitoramento de DoS
  • Monitoramento de rede
  • Informações sobre usuários e aplicativos
  • Tempos de uso de pico
  • Roteamento de tráfego
Importante: O NetFlow não faz uma captura completa de pacote ou captura o conteúdo real. Ele registra informações sobre o fluxo de pacotes, incluindo metadados. A Cisco desenvolveu o NetFlow e depois permitiu que ele fosse usado como base para o padrão IETF chamado IPFIX (baseado no Cisco NetFlow Versão 9).

Atributos do NetFlow (5-tuplas)

1. Endereço IP de origem
2. Endereço IP de destino
3. Porta de origem
4. Porta de destino
5. Tipo de protocolo da camada 3
6. Classe de Serviço (ToS)
7. Interface do roteador ou switch

Todos os pacotes com os mesmos atributos são agrupados em um fluxo. O cache do NetFlow armazena essas informações condensadas. 

# Exemplo de registro NetFlow v5
Date flow start: 2017-08-30
Duration: 0.082899
Proto: TCP
Src IP: 192.168.1.10:52034
Dst IP: 172.16.1.100:80
Flags: ACK
Packets: 389
Bytes: 1495

O sistema Cisco Application Visibility and Control (AVC) combina várias tecnologias para reconhecer, analisar e controlar mais de 1000 aplicativos. Estes incluem voz e vídeo, e-mail, compartilhamento de arquivos, jogos, ponto a ponto (P2P) e aplicativos baseados em nuvem.

20.3.4 Logs de Filtro de Conteúdo

Cisco ESA (Email Security Appliance)

O ESA possui mais de 30 logs que podem ser usados para monitorar:

  • Entrega de e-mail
  • Funcionamento do sistema
  • Antivírus e antispam
  • Decisões de lista negra/branca

Os logs podem ser enviados para servidores syslog, FTP ou SCP.

Cisco WSA (Web Security Appliance)

O WSA atua como um proxy Web, registrando:

  • Informações de transação HTTP
  • Logs de decisão de ACL
  • Logs de varredura de malware
  • Logs de filtragem de reputação Web

Formato compatível com W3C, envio via syslog, FTP ou SCP.

Os dispositivos de segurança Cisco podem ser configurados para enviar eventos e alertas para plataformas de gerenciamento usando SNMP ou syslog.

20.3.7 Cisco Umbrella

O Cisco Umbrella (anteriormente OpenDNS) oferece um serviço DNS hospedado que amplia a capacidade do DNS para incluir aprimoramentos de segurança. Em vez de organizações hospedarem e manterem listas negras, o Cisco Umbrella fornece essas proteções em seu próprio serviço DNS.

Serviços do Umbrella

  • Inteligência contra ameaças em tempo real
  • Gerenciamento de acesso DNS
  • Segurança de registros DNS

20.3.8 Firewalls de Próxima Geração (NGFW)

Os dispositivos de firewall da próxima geração (NextGen) estendem a segurança da rede além dos endereços IP e números de porta da Camada 4 para a camada do aplicativo e além. A linha Cisco Firepower consolida várias camadas de segurança em uma única plataforma.

Serviços do Firepower

  • Visibilidade e controle de aplicativos
  • NGIPS (IPS de próxima geração)
  • Filtragem de URL por categoria
  • Proteção avançada contra malware (AMP)
  • Controle de política de identidade
  • Análise e automação

Tipos de Eventos NGFW

Evento de Conexão

Logs com data/hora, IPs origem/destino, regra de controle de acesso

Evento de Intrusão

Registro de explorações detectadas, origem e destino do ataque

Evento de Host/Endpoint

Detecção de novos hosts na rede, hardware, IP, última presença

Evento de Descoberta de Rede

Alterações detectadas na rede monitorada

Evento de Fluxo de Rede

Registros NetFlow exportados para geração de eventos

Verifique sua compreensão de diferentes tipos de dados de segurança de rede respondendo às perguntas.

1

O que é usado para gerar e visualizar capturas completas de pacotes?

A NetFlow
B tcpdump
C Logs de proxy
D Syslog
Resposta Correta: tcpdump
O tcpdump é uma ferramenta de linha de comando que captura e exibe pacotes de rede em tempo real. O Wireshark é uma GUI construída sobre a funcionalidade do tcpdump. NetFlow registra metadados de fluxo, não o conteúdo completo dos pacotes.
2

Quais dois valores fazem parte de todos os registros de fluxo NetFlow? (Escolha duas.)

A Carimbo de data/hora inicial
B detalhes completos do pacote
C Carimbo de data/hora final
D Requisições de servidor DNS
E ID da aplicação
Resposta Correta: Carimbo de data/hora inicial e Carimbo de data/hora final
Todos os registros de fluxo NetFlow contêm carimbos de data/hora de início e fim do fluxo, além das cinco tuplas (IP origem/destino, portas origem/destino, protocolo). Detalhes completos de pacote não são armazenados pelo NetFlow.
3

O que o AVC (Visibilidade e Controle de Aplicativos) usa para descobrir os aplicativos responsáveis pelo tráfego de rede?

A NBAR2
B captura de pacote completo
C Registros de DNS
D Registros de fluxo NetFlow
E logs de pacotes de firewall
Resposta Correta: NBAR2
O NBAR2 (Network Based Application Recognition versão 2) é o mecanismo de reconhecimento de aplicativos usado pelo Cisco AVC. Ele suporta mais de 1000 aplicativos de rede, identificando-os por assinaturas de tráfego, não apenas por portas.
4

Quais dois dispositivos criarão logs de conteúdo suspeito detectado no tráfego de aplicativos? (Escolha duas.)

A NetFlow
B Dispositivo de segurança de email
C Dispositivo de segurança de web
D tcpdump
E NBAR2
Resposta Correta: Dispositivo de segurança de email (ESA) e Dispositivo de segurança de web (WSA)
O Cisco ESA (Email Security Appliance) e WSA (Web Security Appliance) são dispositivos de filtragem de conteúdo que geram logs detalhados sobre tráfego suspeito, incluindo malware, spam e conteúdo malicioso. NetFlow registra metadados de fluxo, não conteúdo.

Verifique sua compreensão dos eventos do NextGen Firewall respondendo às perguntas.

1

Que tipo de evento ocorre quando ocorrem atividades mal-intencionadas que podem afetar a disponibilidade, integridade e confidencialidade de um host e seus dados?

A Intrusão
B Host ou dispositivo final
C NetFlow
D Configuração
E Descoberta de Rede
F Conexão
Resposta Correta: Intrusão
Eventos de intrusão são gerados quando o sistema identifica atividades mal-intencionadas que podem afetar a disponibilidade, integridade e confidencialidade de um host. O NGIPS (Next-Generation IPS) examina pacotes em busca de explorações conhecidas e gera esses eventos.
2

A Intrusão
B Host ou dispositivo final
C NetFlow

E Descoberta de Rede
F Conexão
Resposta Correta: Host ou dispositivo final
Quando um host aparece pela primeira vez na rede, o sistema NGFW detecta e registra um evento de host/endpoint, incluindo detalhes como hardware do dispositivo, endereçamento IP e a última presença conhecida na rede.
3

Que tipo de evento é usado quando o NetFlow detecta um novo host na rede?

A Intrusão
B Host ou dispositivo final
C NetFlow
D Configuração
E Descoberta de Rede
F Conexão
Resposta Correta: NetFlow
A descoberta de rede pode usar registros de fluxo NetFlow exportados para gerar novos eventos para hosts e servidores. Quando o NetFlow detecta um novo host na rede, um evento de fluxo de rede é registrado.
4

A Intrusão
B Host ou dispositivo final
C NetFlow
D Configuração
E Descoberta de Rede
F Conexão
Resposta Correta: Conexão
Eventos de conexão são logs de sessões detectados diretamente pelo NGIPS. Eles incluem propriedades básicas de conexão como carimbos de data/hora, endereços IP de origem e destino, e metadados sobre qual regra de controle de acesso registrou o evento.
5

Que tipo de evento ocorre quando são detectadas alterações em hosts de rede e aplicativos conhecidos pela rede?

A Intrusão
B Host ou dispositivo final
C NetFlow
D Configuração
E Descoberta de Rede
F Conexão
Resposta Correta: Descoberta de Rede
Eventos de descoberta de rede representam alterações detectadas na rede monitorada. Essas alterações são registradas em resposta às diretivas de descoberta de rede que especificam os tipos de dados a serem coletados e os segmentos de rede a serem monitorados.

20.4 Resumo de Dados de Segurança de Rede

Consolidação dos principais conceitos abordados neste módulo sobre tipos de dados de segurança, logs de dispositivos e logs de rede.

20.4.1 O que aprendi neste módulo?

Tipos de Dados de Segurança

Dados de Alerta

Mensagens geradas por IPS/IDS (Snort) quando o tráfego viola regras ou corresponde a assinaturas de explorações. Visualizados em Sguil, Squert e Kibana.

Dados de Sessão

Registro de conversas entre dois pontos de rede. Inclui cinco tuplas (IPs e portas origem/destino, protocolo), ID de sessão, quantidade de dados transferidos e duração.

Capturas Completas de Pacotes

Dados mais detalhados. Contêm o conteúdo real das conversas: texto de emails, HTML, arquivos transferidos. Maior requisito de armazenamento.

Dados Estatísticos

Criados através da análise de outras formas de dados. Permitem criar baseline do comportamento normal e detectar anomalias estatisticamente significativas.

Conteúdo Extraído

Recuperado de capturas completas de pacotes. Arquivos, anexos e objetos transferidos podem ser analisados em sandboxes para detecção de malware.

Dados de Transação

Mensagens trocadas durante sessões. Incluem logs de servidores web (acesso/erro) e registros de aplicações.

Registros de Dispositivos Finais

HIDS (Host-based IDS)

Executam em hosts individuais, detectam e previnem intrusões. OSSEC é um HIDS de código aberto popular com funcionalidade robusta de coleta e análise de logs.

Syslog

Padrão para formatos de mensagem, estrutura cliente-servidor e protocolo de rede. Dispositivos de rede enviam eventos para servidores syslog centralizados.

Logs de Servidor

Servidores de email e Web mantêm registros de acesso e erros. Logs de proxy DNS documentam consultas DNS, úteis para identificar exfiltração e conexões CNC.

SIEM

Combina SEM (Security Event Management) e SIM (Security Information Management) para visão abrangente usando coleta de logs, normalização, correlação, agregação e relatórios.

Logs de Rede

tcpdump

Analisador de pacotes em linha de comando. Exibe capturas em tempo real ou grava em arquivo. Wireshark é a GUI para visualização.

NetFlow

Formato baseado em sessão para contabilidade de tráfego, faturamento, planejamento de rede, segurança e monitoramento. Fornece informações sobre usuários, aplicativos e roteamento.

AVC (Application Visibility and Control)

Usa NBAR2 para descobrir e classificar mais de 1000 aplicativos em uso na rede, incluindo voz, vídeo, email, P2P e aplicativos em nuvem.

ESA e WSA

Email Security Appliance e Web Security Appliance fornecem filtragem de conteúdo e logs detalhados para monitoramento de segurança.

Servidores Proxy

Atuam como intermediários para clientes. Geram logs de solicitações e respostas para análise de destinos seguros, downloads e comportamento de usuários.

NextGen Firewall (NGFW)

Estendem segurança para camada de aplicativo. Painéis interativos permitem relatórios rápidos sem necessidade de SIEM. Incluem Firepower Services.

Tipo de Dado Descrição Ferramentas/Exemplos

👥 Casos de Uso: Como isso funciona no dia a dia

🕵️ O Detetive de Rede

Dados de Sessão vs Captura Completa

História: Imagine que você é um detetive investigando um roubo em um prédio comercial.

Analogia:
  • Dados de Sessão = Registro de entrada e saída do prédio. Você vê quem entrou, quando, por onde e quanto tempo ficou.
  • Captura Completa (PCAP) = As câmeras de segurança. Você vê exatamente o que a pessoa feu, o que carregava e com quem falou.
Na prática: Se alguém baixou um arquivo suspeito, a sessão mostra que houve download; a captura completa mostra qual arquivo foi baixado.

🚪 O Porteiro Inteligente

NGFW e Controle de Aplicativos

História: Um condomínio tem um porteiro que conhece todos os moradores e visitantes habituais.

Analogia:
  • Firewall Tradicional = Porteiro que só olha a identidade (IP e porta).
  • NGFW (AVC/NBAR2) = Porteiro que reconhece o que você veio fazer. Sabe diferenciar um entregador (Netflix) de um assaltante (malware).
Na prática: O NGFW bloqueia Netflix no horário comercial, mas permite videoconferências importantes, mesmo usando a mesma porta (443).

🧠 A Central de Inteligência

SIEM e Correlação de Eventos

História: Uma empresa percebeu que algo estava errado, mas não conseguia identificar o problema.

Analogia:
  • Separadamente: Firewall viu tentativas de login estranhas.
  • Servidor registrou acesso de madrugada.
  • Antivírus detectou algo suspeito.
  • SIEM juntou tudo e disse: "TEM UM INVASOR!"
Na prática: Sozinhos, os alertas parecem inofensivos. Juntos, contam a história completa de um ataque.

📬 O Carteiro que Filtra

ESA / WSA - Filtragem de Conteúdo

História: Uma empresa queria evitar que funcionários abrissem emails e sites perigosos.

Analogia:
  • ESA (Email) = Um carteiro que abre todas as cartas antes de entregar. Se encontrar algo perigoso, devolve ou descarta.
  • WSA (Web) = Um segurança que verifica todos os pacotes antes de deixar você entrar em um site.
Na prática: Um funcionário recebe um email com link suspeito. O WSA bloqueia o acesso e o ESA registra o incidente para investigação.

🗺️ O Mapa do Crime

NetFlow e Detecção de Anomalias

História: Um analista de segurança notou um padrão estranho no tráfego da rede.

Analogia:
  • Normal: Na sexta à noite, o tráfego da rede cai (todo mundo foi embora).
  • Anomalia: De repente, um computador começa a enviar terabytes de dados à meia-noite.
  • NetFlow = Mostra que esse computador está se comunicando com um servidor na Rússia.
Na prática: É assim que se descobre exfiltração de dados - um funcionário ou malware roubando informações fora do horário comercial.

☁️ A Lista Negra Automática

Cisco Umbrella - DNS como Segurança

História: Um funcionário tentou acessar um site de phishing, mas foi bloqueado antes mesmo de carregar.

Analogia:
  • Funcionário digita site-falso.com
  • O computador pergunta ao Umbrella: "Onde fica esse site?"
  • Umbrella responde: "Esse lugar é perigoso! Vou te redirecionar para uma página de aviso."
Na prática: O bloqueio acontece antes mesmo de qualquer dado ser trocado, protegendo o usuário antes do contato com o site malicioso.

🏙️ A Metáfora da Cidade Digital

Imagine que sua rede é uma cidade com prédios (servidores), casas (computadores) e ruas (rede).

tcpdump/Wireshark = Câmeras de segurança gravando tudo que acontece nas ruas.
Dados de Sessão (NetFlow) = Registro de quem entrou na cidade, quando e por quanto tempo.
IDS/IPS (Snort) = Guardas que patrulham e gritam quando veem alguém suspeito.
SIEM = Central de inteligência que junta relatórios de todos os guardas e câmeras.
ESA/WSA = Porteiros que inspecionam encomendas (emails) e visitantes (sites).
NGFW = Portaria inteligente que reconhece o que cada visitante veio fazer.
Dados Estatísticos = Relatório que mostra que às 18h o tráfego aumenta (todo mundo voltando do trabalho).
Cisco Umbrella = Mapa da cidade que já marca lugares perigosos para ninguém entrar.

✅ Checklist do Analista: O que observar em cada tipo de dado

Dados de Alerta Alguma regra do Snort foi acionada? Qual a gravidade?
Dados de Sessão Um host está se comunicando com IPs estranhos? Há tráfego em horários incomuns?
Captura Completa O que realmente foi transferido? Há arquivos suspeitos?
Dados Estatísticos O tráfego atual está dentro da baseline normal?
Logs ESA/WSA Alguém recebeu emails com anexos perigosos? Tentou acessar sites bloqueados?
Eventos NGFW Houve eventos de intrusão? Novos hosts apareceram na rede?

❓ Perguntas Frequentes (FAQ)

Qual a diferença entre IDS e IPS?

IDS (Intrusion Detection System) só alerta sobre ameaças. IPS (Intrusion Prevention System) bloqueia ativamente. É como um segurança que só observa (IDS) vs um que intervém (IPS).

Por que não usar só captura completa de pacotes?

Porque ocupa MUITO espaço. Uma hora de tráfego pode gerar gigabytes de dados. NetFlow e dados de sessão são muito mais leves e permitem armazenar meses de histórico.

O que é mais importante: SIEM ou NGFW?

Ambos! NGFW é a defesa (portaria), SIEM é a inteligência (central de investigação). Um sem o outro deixa brechas.

NetFlow substitui Wireshark?

Não! NetFlow mostra quem falou com quem e quanto dados trocaram. Wireshark mostra o que foi dito. Um complementa o outro.

Como saber se um alerta é falso positivo?

Analise o contexto: é um comportamento normal? O host envolvido é crítico? Use dados de sessão para verificar se há padrões suspeitos adicionais.

Preciso de tudo isso em casa?

Para uso doméstico, um bom antivírus e firewall já bastam. Essas ferramentas são para empresas e profissionais que precisam proteger redes complexas.
