20.1 Tipos de Dados de Segurança

20.1.1 Dados de Alerta

Os dados de alerta consistem em mensagens geradas por sistemas de prevenção de intrusões (IPSs) ou sistemas de detecção de intrusões (IDSs) em resposta ao tráfego que viola uma regra ou corresponde à assinatura de uma exploração conhecida.

Um IDS de rede (NIDS), como o Snort, vem configurado com regras para explorações conhecidas. Os alertas são gerados pelo Snort e são legíveis e pesquisáveis pelos aplicativos Sguil e Squert, que fazem parte do conjunto Security Onion de ferramentas NSM. 

# Regra Snort para detecção
alert ip any any -> any any (msg:"GPL ATTACK_RESPONSE id check returned root"; 
content:"uid=0|28|root|25"; sid:2100498; rev:8;)

Site de teste: testmyds.com

Retorna o texto: uid=0(root) gid=0(root) groups=0(root)

Snort Sguil Security Onion
Importante: O site testmyds é uma ferramenta inofensiva usada para verificar se o NIDS está funcionando corretamente. Quando um host visita este site, uma assinatura é correspondida e um alerta é acionado.

20.1.2 Dados de Sessão

Os dados de sessão são um registro de uma conversa entre dois pontos de extremidade de rede, que geralmente são um cliente e um servidor. Dados de sessão são dados sobre a sessão, não os dados recuperados e usados pelo cliente.

Campos do Dado de Sessão (Zeek)

1. ts: carimbo de data e hora
2. uid: ID de sessão única
3. id_orig_h: IP de origem
4. id_orig_p: porta de origem
5. id_resp_h: IP de destino
6. id_resp_p: porta de destino
7. proto: protocolo de transporte
8. service: protocolo de aplicação
9. duration: duração da sessão
10. orig_bytes: bytes do host de origem
11. resp_bytes: bytes do host respondente
12. orig_packets: pacotes do host de origem
13. resp_packets: pacotes do host respondente

As Cinco Tuplas

  • IP Origem
  • Porta Origem
  • IP Destino
  • Porta Destino
  • Protocolo

Zeek (anteriormente Bro) é uma ferramenta de monitoramento de segurança de rede.

20.1.3 Dados de Transação

Os dados de transação consistem nas mensagens que são trocadas durante sessões de rede. Essas transações podem ser exibidas em transcrições de captura de pacotes. Os logs de dispositivos mantidos por servidores também contêm informações sobre as transações que ocorrem entre clientes e servidores. 

# Requisição HTTP
GET /home/index.html HTTP/1.1
Host: www.example.com
User-Agent: Mozilla/5.0 (Windows NT 6.1; WOW64; rv:53.0)

# Resposta HTTP
HTTP/1.1 200 OK
Date: Fri, 10 Oct 2015 23:59:59 GMT
Content-Type: text/plain

<text returned>
# Log de Acesso
192.168.1.10 - anyUser [10/Oct/2015:13:55:36 -0500] 
"GET /index.html HTTP/1.1" 200 326
Diferença: A sessão é todo o tráfego envolvido na elaboração da solicitação; a transação é a própria solicitação.

20.1.4 Capturas Completas de Pacotes

Capturas completas de pacotes são os dados de rede mais detalhados que geralmente são coletados. Devido à quantidade de detalhes, eles também são os tipos de dados mais intensos de armazenamento e recuperação.

As capturas completas de pacotes contêm não apenas dados sobre conversas de rede, como dados de sessão, mas também o conteúdo real das conversas:

  • Texto das mensagens de email
  • HTML nas páginas da Web
  • Arquivos que entram ou saem da rede

A ferramenta familiar Wireshark é muito popular para visualizar capturas de pacotes completos.

20.1.5 Dados Estatísticos

Os dados estatísticos são criados através da análise de outras formas de dados de rede. Podem ser feitas conclusões que descrevem ou predizem o comportamento da rede a partir dessas análises.

NBA / NBAD

Network Behavior Analysis e Network Behavior Anomaly Detection usam técnicas analíticas avançadas para analisar dados NetFlow ou IPFIX.

Análise Preditiva IA / Machine Learning Detecção de Anomalias
Cisco Cognitive Threat Analytics

Produto baseado em nuvem que usa aprendizado de máquina e modelagem estatística para:

  • Criar linha de base do tráfego
  • Identificar anomalias
  • Analisar comportamento de usuários e dispositivos
  • Detectar comunicação CNC e exfiltração
# Fluxo de Análise Estatística
Tráfego de Rede → Coleta de Dados → Análise Estatística → 
Linha de Base → Detecção de Anomalias → Alerta

20.2 Logs de Dispositivos Finais

Os dispositivos finais (endpoints) geram logs que são essenciais para o monitoramento de segurança. Estes incluem:

  • Logs de Sistema: Eventos do sistema operacional
  • Logs de Aplicações: Atividades de software instalado
  • Logs de Segurança: Tentativas de login, alterações de privilégios
  • Logs de Autenticação: Sucessos e falhas de autenticação
Importante: A correlação entre logs de endpoints e logs de rede permite identificar a origem e o impacto de incidentes de segurança. 
# Exemplo de Log de Autenticação (Linux)
Mar 31 15:30:22 servidor sshd[12345]: Failed password for user admin from 192.168.1.100 port 54321 ssh2
Mar 31 15:30:25 servidor sshd[12345]: Accepted password for user admin from 192.168.1.100 port 54322 ssh2

# Exemplo de Log de Sistema (Windows)
Event ID 4624: Account Successfully Logged On
Event ID 4625: Account Failed to Log On

20.3 Logs de Rede

Os logs de rede são coletados de diversos dispositivos de infraestrutura:

  • Roteadores e Switches: Logs de interface, erros de roteamento
  • Firewalls: Conexões permitidas/negadas, NAT
  • Servidores DNS: Consultas de resolução de nomes
  • Servidores DHCP: Atribuição de endereços IP
  • NetFlow/IPFIX: Dados de fluxo de rede
Ferramentas comuns: Syslog-ng, ELK Stack (Elasticsearch, Logstash, Kibana), Splunk, Security Onion

20.2.5 Verifique sua compreensão - Identifique os eventos de segurança do Windows

1

Qual nível de segurança do Windows é registrado quando um logon remoto foi tentado sem êxito por um usuário não autorizado?

A Erro
B Fatal
C Aviso
D Informações
E Sucesso na Auditoria
F Falha na Auditoria
Resposta Correta: Falha na Auditoria
Eventos de "Falha na Auditoria" são registrados quando uma tentativa de logon ou acesso falha, especialmente quando um usuário não autorizado tenta acessar o sistema. Isso é crucial para detectar tentativas de invasão e ataques de força bruta. O Event ID 4625 no Windows registra falhas de logon.
2

Qual nível de segurança do Windows é registrado quando um processo necessário foi carregado com êxito em uma estação de trabalho?

A Erro
B Fatal
C Aviso
D Informações
E Sucesso na Auditoria
F Falha na Auditoria
Resposta Correta: Informações
O nível "Informações" é usado para registrar eventos normais e bem-sucedidos do sistema, como a inicialização de serviços, carregamento de processos e outras operações que ocorrem conforme o esperado. Esses eventos ajudam a estabelecer uma linha de base do comportamento normal do sistema.
3

Qual nível de segurança do Windows é registrado quando ocorreu um erro de disco durante uma operação de backup?

A Erro
B Fatal
C Aviso
D Informações
E Sucesso na Auditoria
F Falha na Auditoria
Resposta Correta: Erro
O nível "Erro" é usado para registrar problemas significativos que afetam a funcionalidade do sistema, como falhas de hardware, erros de disco ou serviços que não conseguem iniciar. Esses eventos requerem atenção do administrador, pois podem indicar problemas que precisam ser resolvidos.
4

Qual nível de segurança do Windows é registrado quando o espaço em disco está ficando baixo?

A Erro
B Fatal
C Aviso
D Informações
E Sucesso na Auditoria
F Falha na Auditoria
Resposta Correta: Aviso
O nível "Aviso" é usado para registrar condições que não são imediatamente críticas, mas podem se tornar problemas se não forem tratadas. Espaço em disco baixo, falhas de driver recuperáveis ou configurações de segurança não ideais são exemplos típicos de eventos de aviso.

A Erro
B Fatal
C Aviso
D Informações

O nível "Sucesso na Auditoria" registra eventos de segurança bem-sucedidos, como logons autorizados, alterações de privilégios e acessos a recursos. Isso é essencial para rastrear atividades legítimas e estabelecer uma trilha de auditoria. O Event ID 4624 no Windows registra logons bem-sucedidos.

20.3 Logs de Rede

20.3.1 Tcpdump

A ferramenta de linha de comando tcpdump é um analisador de pacotes muito popular. Ele pode exibir capturas de pacotes em tempo real ou gravar capturas de pacotes em um arquivo. Ele captura dados detalhados de protocolo de pacotes e conteúdo.

Wireshark é uma GUI construída sobre a funcionalidade tcpdump. A estrutura das capturas tcpdump varia dependendo do protocolo capturado e dos campos solicitados. 

# Comandos básicos do tcpdump
tcpdump -i eth0                    # Capturar na interface eth0
tcpdump -w captura.pcap            # Salvar captura em arquivo
tcpdump -r captura.pcap            # Ler arquivo de captura
tcpdump port 80                    # Filtrar por porta
tcpdump host 192.168.1.1           # Filtrar por endereço IP
tcpdump -n                         # Não resolver nomes DNS

tcpdump → Wireshark

Análise de pacotes em tempo real e offline

CLI GUI PCAP

20.3.2 NetFlow

NetFlow é um protocolo desenvolvido pela Cisco como uma ferramenta para solução de problemas de rede e contabilidade baseada em sessão. O NetFlow fornece com eficiência um importante conjunto de serviços para aplicativos IP, incluindo:

  • Contabilidade de tráfego de rede
  • Faturamento de rede com base no uso
  • Planejamento de rede
  • Segurança e monitoramento de DoS
  • Monitoramento de rede
  • Informações sobre usuários e aplicativos
  • Tempos de uso de pico
  • Roteamento de tráfego
Importante: O NetFlow não faz uma captura completa de pacote ou captura o conteúdo real. Ele registra informações sobre o fluxo de pacotes, incluindo metadados. A Cisco desenvolveu o NetFlow e depois permitiu que ele fosse usado como base para o padrão IETF chamado IPFIX (baseado no Cisco NetFlow Versão 9).

Atributos do NetFlow (5-tuplas)

1. Endereço IP de origem
2. Endereço IP de destino
3. Porta de origem
4. Porta de destino
5. Tipo de protocolo da camada 3
6. Classe de Serviço (ToS)
7. Interface do roteador ou switch

Todos os pacotes com os mesmos atributos são agrupados em um fluxo. O cache do NetFlow armazena essas informações condensadas. 

# Exemplo de registro NetFlow v5
Date flow start: 2017-08-30
Duration: 0.082899
Proto: TCP
Src IP: 192.168.1.10:52034
Dst IP: 172.16.1.100:80
Flags: ACK
Packets: 389
Bytes: 1495

20.3.4 Logs de Filtro de Conteúdo

Cisco ESA (Email Security Appliance)

O ESA possui mais de 30 logs que podem ser usados para monitorar:

  • Entrega de e-mail
  • Funcionamento do sistema
  • Antivírus e antispam
  • Decisões de lista negra/branca

Os logs podem ser enviados para servidores syslog, FTP ou SCP.

Cisco WSA (Web Security Appliance)

O WSA atua como um proxy Web, registrando:

  • Informações de transação HTTP
  • Logs de decisão de ACL
  • Logs de varredura de malware
  • Logs de filtragem de reputação Web

Formato compatível com W3C, envio via syslog, FTP ou SCP.

20.3.7 Cisco Umbrella

O Cisco Umbrella (anteriormente OpenDNS) oferece um serviço DNS hospedado que amplia a capacidade do DNS para incluir aprimoramentos de segurança. Em vez de organizações hospedarem e manterem listas negras, o Cisco Umbrella fornece essas proteções em seu próprio serviço DNS.

Serviços do Umbrella

  • Inteligência contra ameaças em tempo real
  • Gerenciamento de acesso DNS
  • Segurança de registros DNS
  • Proteção contra phishing
  • Filtragem de conteúdo por categoria

20.3.8 Firewalls de Próxima Geração (NGFW)

Os dispositivos de firewall da próxima geração (NextGen) estendem a segurança da rede além dos endereços IP e números de porta da Camada 4 para a camada do aplicativo e além. A linha Cisco Firepower consolida várias camadas de segurança em uma única plataforma.

Serviços do Firepower

  • Visibilidade e controle de aplicativos
  • NGIPS (IPS de próxima geração)
  • Filtragem de URL por categoria
  • Proteção avançada contra malware (AMP)
  • Controle de política de identidade
  • Análise e automação

Tipos de Eventos NGFW

Evento de Conexão

Logs com data/hora, IPs origem/destino, regra de controle de acesso

Evento de Intrusão

Registro de explorações detectadas, origem e destino do ataque

Evento de Host/Endpoint

Detecção de novos hosts na rede, hardware, IP, última presença

Evento de Descoberta de Rede

Alterações detectadas na rede monitorada

Evento de Fluxo de Rede

Registros NetFlow exportados para geração de eventos
