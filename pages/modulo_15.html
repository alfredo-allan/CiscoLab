15.0 Tecnologias de Firewall

Protegendo redes através de políticas de controle de acesso e arquiteturas de segurança.

15.0.1 Por que fazer este módulo?

Com as muitas ameaças à segurança da rede, as redes precisam ser projetadas para proteger recursos de dados e garantir que os serviços de rede sejam fornecidos conforme necessário. A infraestrutura de segurança de rede define a forma como os dispositivos são conectados entre si para obter comunicações seguras de ponta a ponta.

Assim como existem muitos tamanhos de redes, também existem muitas maneiras de construir uma infraestrutura de rede segura. No entanto, existem projetos padrão que a indústria recomenda para obter redes disponíveis e seguras. Este módulo aborda as maneiras básicas pelas quais os firewalls são usados para criar uma arquitetura de segurança de rede.

Importante: A falta de proteção contra ataques de malware pode comprometer a integridade dos sistemas e a segurança de dados.

15.1 Proteger redes com firewalls

15.1.1 Firewalls

Operação de firewall

Um firewall é um sistema ou grupo de sistemas que aplica uma política de controle de acesso entre redes.

Operação do Firewall

REDE EXTERNA

Internet / Não confiável

FIREWALL

Controle de acesso
✓ Permitir tráfego Web (porta 80/443) ✓ Permitir tráfego FTP (porta 21) ✗ Negar ICMP echo requests ✗ Negar tráfego de endereços suspeitos

REDE INTERNA

Corporativa / Confiável
Permitir tráfego de qualquer endereço externo para o servidor Web

O firewall atua como um ponto de controle entre redes confiáveis e não confiáveis, aplicando regras que determinam qual tráfego é permitido ou negado com base em critérios como endereço IP, porta e protocolo.

Propriedades comuns do Firewall

Todos os firewalls compartilham algumas propriedades comuns:

Resistência a ataques

Os firewalls são resistentes a ataques de rede, projetados para operar sob condições adversas.

Único ponto de trânsito

Todo o tráfego entre redes internas e externas flui através do firewall.

Reforça políticas

Os firewalls aplicam a política de controle de acesso definida pela organização.

Benefícios do firewall

Proteção de recursos sensíveis

Impedem a exposição de hosts, recursos e aplicações sensíveis a usuários não confiáveis.

Sanitização de protocolos

Impedem a exploração de falhas nos protocolos de rede.

Bloqueio de dados maliciosos

Bloqueiam dados maliciosos direcionados a servidores e clientes.

Simplificação do gerenciamento

Reduz a complexidade do gerenciamento de segurança, centralizando o controle de acesso.

Limitações do Firewall

Configuração inadequada

Um firewall mal configurado pode se tornar um único ponto de falha na rede.

Limitações de aplicativos

Dados de muitos aplicativos não podem ser transmitidos por firewalls com segurança.

Contorno por usuários

Usuários podem tentar contornar o firewall para acessar material bloqueado.

Impacto no desempenho

O desempenho da rede pode diminuir devido à inspeção de pacotes.

Importante: Tráfego não autorizado pode ser encapsulado ou escondido como tráfego legítimo através do firewall.

15.1.2 Tipos de firewalls

É importante entender os diferentes tipos de firewalls e suas capacidades específicas para que o firewall correto seja usado para cada situação.

Firewall de filtragem de pacotes (sem estado) Firewall com monitoração de estado Firewall de gateway de aplicativo Firewall de última geração
Opera nas camadas 3 e 4 Mantém tabela de estado Proxy de aplicações IPS integrado + controle de apps

Firewall de filtragem de pacotes (sem estado)

Os firewalls de filtragem de pacotes geralmente fazem parte de um firewall de roteador, que permite ou nega tráfego com base nas informações da Camada 3 (Rede) e Camada 4 (Transporte).

Eles são firewalls sem estado (stateless) que usam uma simples pesquisa de tabela de políticas que filtra o tráfego com base em critérios específicos, como:

  • Endereço IP de origem/destino
  • Porta de origem/destino
  • Protocolo (TCP, UDP, ICMP)
Exemplo: Servidores SMTP escutam a porta 25 por padrão.
Um administrador pode configurar o firewall para bloquear a porta 25 
de uma estação de trabalho específica para evitar que ela transmita um vírus de e-mail.
Camada 3 - Rede Camada 4 - Transporte Camadas superiores: ❌ Não inspeciona

Firewall com monitoração de estado (Stateful)

Firewalls com estado são as tecnologias de firewall mais versáteis e mais comuns em uso. Eles fornecem filtragem de pacotes stateful usando informações de conexão mantidas em uma tabela de estado.

Filtragem com estado é uma arquitetura de firewall classificada na camada de rede. Ele também analisa o tráfego nas camadas 4 e 5 do modelo OSI.

Tabela de Estado
OrigemDestinoEstado
192.168.1.10:543218.8.8.8:443ESTABLISHED
10.0.0.5:12345203.0.113.1:80ESTABLISHED
Benefícios
  • Melhor desempenho que proxies
  • Maior segurança que filtragem simples
  • Rastreia o estado das conexões
  • Permite apenas tráfego relacionado a conexões iniciadas internamente
Camada 3 - Rede Camada 4 - Transporte Camada 5 - Sessão

Firewall de gateway de aplicativo (Firewall Proxy)

Um firewall de gateway de aplicação (firewall proxy), filtra as informações nas camadas 3, 4, 5 e 7 do modelo de referência OSI. A maior parte do controle e filtragem do firewall é feita em software.

Cliente

Servidor Proxy

Servidor Remoto

O servidor proxy se conecta ao servidor remoto em nome do cliente. O servidor remoto só vê uma conexão do servidor proxy.

Características
  • Inspeção profunda de pacotes (DPI)
  • Autenticação de usuários
  • Cache de conteúdo
  • Oculta a estrutura da rede interna
Camada 3 - Rede Camada 4 - Transporte Camada 5 - Sessão Camada 7 - Aplicação

Firewall de última geração (NGFW)

Os firewalls de última geração (NGFW - Next Generation Firewall) vão além dos firewalls de estado, fornecendo funcionalidades avançadas de segurança.

Prevenção de intrusão integrada (IPS)

Reconhecimento e controle de aplicativos

Atualizações contínuas de inteligência de ameaças

Técnicas avançadas contra ameaças evoluídas

Exemplo de firewall de última geração da Cisco

Outras implementações de firewalls

Firewall baseado em host

PC ou servidor com software de firewall em execução nele. Ex: Windows Defender Firewall.

Firewall transparente

Filtra o tráfego IP entre um par de interfaces em ponte, sem exigir configuração de gateway.

Firewall híbrido

Combinação dos vários tipos de firewall. Ex: firewall de inspeção de aplicações combina stateful com gateway de aplicativo.

15.2 Firewalls em Design de Rede

15.2.1 Arquiteturas comuns de segurança

O design do firewall é principalmente sobre interfaces de dispositivo que permitem ou negam tráfego com base na origem, no destino e no tipo de tráfego. Alguns designs são tão simples quanto designar uma rede externa e uma rede interna, que são determinadas por duas interfaces em um firewall.

Aqui estão três designs comuns de firewall:

Privado e Público

Privado
Confiável

Firewall

Público
Não confiável

  • Tráfego interno → externo: permitido e inspecionado
  • Tráfego de retorno: permitido
  • Tráfego externo → interno: bloqueado

Zona Desmilitarizada (DMZ)

DMZ

Privado

Firewall

Público

  • Privado → DMZ/Público: permitido
  • DMZ → Privado: bloqueado
  • Público → DMZ: permitido seletivamente (web, email, DNS)
  • Público → Privado: bloqueado

Firewalls de política baseados em zona (ZPF)

Zona LAN 1 Zona LAN 2 Zona Internet
  • Uma zona = grupo de interfaces com funções semelhantes
  • Tráfego dentro da mesma zona: permitido livremente
  • Tráfego entre zonas: bloqueado por padrão
  • Políticas configuradas para permitir tráfego entre zonas

Auto Zona: O próprio roteador (SSH, SNMP, protocolos de roteamento)

15.2.2 Defesa em camadas (Defense in Depth)

Uma defesa em camadas usa diferentes tipos de firewalls e medidas de segurança que são combinadas em diferentes áreas da rede para adicionar profundidade à segurança de uma organização.

1. Segurança do núcleo da rede

Protege contra software malicioso e anomalias de tráfego, impõe políticas de rede e garante capacidade de sobrevivência

2. Segurança de perímetro

Protege limites entre zonas (firewalls, IDS/IPS)

3. Segurança das comunicações

Fornece garantia de informação (VPN, TLS/SSL)

4. Segurança do endpoint

Fornece identidade e conformidade com a política de segurança do dispositivo

As políticas podem ser aplicadas em camadas ou dentro das camadas. Esses sistemas podem ser configurados para garantir proteção de dados, controle de acesso e integridade de serviços.

Firewall de Host

Firewall interno localizado na máquina física do computador. Protege os recursos internos do servidor.

Ex: Windows Defender Firewall
Firewall de Servidor

Firewall interno localizado na máquina virtual do servidor. Protege os recursos externos do servidor.

Ex: Firewall em ambiente virtualizado

Práticas recomendadas para políticas de segurança de firewall

  • Posicione firewalls nos limites de segurança
  • Negue todo o tráfego por padrão
  • Permitir apenas serviços que são necessários
  • Assegure-se de que o acesso físico ao firewall esteja controlado
  • Monitore regularmente logs de firewall
  • Pratique o gerenciamento de alterações para configurações
  • Lembre-se: firewalls protegem contra ataques técnicos originários do exterior
  • Não confie exclusivamente em um firewall para segurança

15.3.1 O que eu aprendi neste módulo?

🛡️ Proteger redes com firewalls

Existem vários tipos diferentes de firewalls. Os firewalls de filtragem de pacotes (sem estado) fornecem filtragem de Camada 3 e, por vezes, Camada 4. Um firewall de inspeção com estado permite ou bloqueia o tráfego com base no estado, na porta e no protocolo. Firewalls de gateway de aplicativo (firewall proxy) filtram informações nas Camadas 3, 4, 5 e 7. Os firewalls de próxima geração (NGFW) fornecem serviços adicionais além dos gateways de aplicações, como prevenção integrada de intrusões, reconhecimento e controle de aplicações para ver e bloquear aplicativos arriscados, acesso a futuros feeds de informações e técnicas para lidar com ameaças de segurança em evolução.

🏗️ Firewalls em projetos de rede

Arquiteturas de segurança comuns definem os limites do tráfego de entrada e saída da rede. Ao examinar uma topologia que tem acesso a redes externas ou públicas, você deve ser capaz de determinar a arquitetura de segurança. Alguns projetos são tão simples como designar uma rede externa e uma rede interna que são determinados por duas interfaces em um firewall.

As redes que exigem acesso público aos serviços geralmente incluem uma DMZ que o público pode acessar, enquanto bloqueia estritamente o acesso à rede interna. ZPFs (Zone-Based Policy Firewalls) usam o conceito de zonas para fornecer flexibilidade adicional. Uma zona é um grupo de uma ou mais interfaces que têm funções, recursos e requisitos de segurança semelhantes.

Uma abordagem de segurança em camadas (Defense in Depth) usa firewalls e outras medidas de segurança para fornecer segurança em diferentes camadas funcionais da rede: núcleo da rede, perímetro, comunicações e endpoints.

🎓 Entendendo Firewalls de Forma Simples

📖 Imagine que sua rede é uma casa...

Se você tem uma casa, você coloca portas e janelas. Mas você também coloca trancas, cadeados e talvez um porteiro para decidir quem entra e quem sai. O firewall é exatamente isso para o seu computador e sua rede!

📜 A História dos Firewalls: Como tudo começou?

Década de 1980
🌅 O Nascimento da Internet

Nos anos 80, a internet começava a se popularizar fora das universidades e militares. Os primeiros computadores se conectavam, mas não havia nenhum controle de quem entrava ou saía. Era como uma casa sem portas - qualquer um podia entrar!

O primeiro grande ataque de vírus (Morris Worm, 1988) infectou 10% de todos os computadores da internet!
1990 - 1994
🚪 Os Primeiros Firewalls - "Filtragem de Pacotes"

Os primeiros firewalls eram como "porteiros" que olhavam apenas o endereço de quem batia na porta. Se o endereço estava em uma lista negra, eles bloqueavam. Mas eles não verificavam o que a pessoa estava trazendo dentro da mala!

✓ Verifica: Endereço IP ✓ Verifica: Porta (80, 443, 25...) ✗ Não verifica: Conteúdo dos dados
1994 - 2000
📝 Firewalls com Estado (Stateful) - "O Porteiro que Anota"

Agora o porteiro começou a anotar quem entrou e quem saiu. Se alguém de dentro da casa mandou um parente buscar um documento, o porteiro deixava esse parente entrar porque sabia que ele era esperado. Isso é o "estado" da conexão - o firewall lembra das conversas que estão acontecendo.

"Se a conversa começou lá dentro, pode entrar. Se começou lá fora, bloqueia!"
2000 - 2010
🔍 Firewalls Proxy - "O Porteiro que Revista"

O porteiro não apenas anota quem entra, mas também revista as malas! Ele abre cada pacote, olha o conteúdo, verifica se é seguro. Só depois ele entrega para o dono da casa. É mais seguro, mas mais lento - como passar pela segurança do aeroporto.

"Vou abrir tudo que entra para ver se não tem nada perigoso!"
2010 - Hoje
🤖 NGFW - Firewalls Inteligentes

Os firewalls de hoje são como "porteiros inteligentes com IA". Eles não só revistam, mas também:

  • Detectam comportamentos suspeitos (alguém tentando arrombar a porta várias vezes)
  • Sabem qual programa está tentando se comunicar (não só o endereço)
  • Aprendem com novos ataques e se atualizam automaticamente

📬 O Modelo OSI - Como uma Carta é Entregue

O modelo OSI é como o processo de enviar uma carta pelo correio. Cada camada tem uma função específica:

Camada 7
Aplicação

📝 O que você quer dizer - É a carta em si. Ex: HTTP (navegador), SMTP (email), FTP (transferir arquivos)

Camada 6
Apresentação

🔒 Como você escreve - Traduz, comprime e criptografa. Ex: converter texto para binário, compactar imagem

Camada 5
Sessão

🤝 Organizando a conversa - Inicia, mantém e encerra a comunicação. Ex: "Vamos conversar?", "Ok!", "Até logo!"

Camada 4
Transporte

🚚 O caminhão que entrega - Garante que a carta chegue inteira. Ex: TCP (confiável, confirma recebimento) ou UDP (rápido, mas pode perder pacotes)

Camada 3
Rede

🗺️ O endereço de entrega - Define o caminho. Ex: Endereço IP (192.168.1.10) - como o CEP da casa

Camada 2
Enlace de Dados

🔌 O carteiro do bairro - Entrega dentro da mesma rede local. Ex: Endereço MAC (físico da placa de rede)

Camada 1
Física

⚡ O meio de transporte - Cabos, Wi-Fi, fibra ótica. Como o caminhão, o avião ou o carteiro a pé

Mnemonico para lembrar as camadas:

"Por favor, Não Tenha Dúvidas, Senhor Antônio Amigo"

(Física, Enlace, Rede, Transporte, Sessão, Apresentação, Aplicação)

🎭 Os Tipos de Firewall - Em Metáforas

Stateless

📋 "O Porteiro com Lista"

Só olha o nome na lista: "João? Pode entrar. Maria? Bloqueada." Não lembra de conversas anteriores.

Camadas 3-4
Stateful

🧠 "O Porteiro que Anota"

Lembra: "Ah, a Maria saiu há pouco, ela está voltando. Pode entrar!" Mantém uma "memória" das conversas.

Camadas 3-5
Proxy

🔍 "O Porteiro que Revista"

Abre a mala, verifica o conteúdo, decide se é seguro. Só depois entrega. Mais seguro, mas mais demorado.

Camadas 3-7
NGFW

🤖 "O Porteiro Robô"

Aprende com ataques, detecta comportamentos suspeitos, sabe qual programa está se comunicando. Inteligente!

Camadas 3-7 + IPS

📚 Glossário - Termos Técnicos Explicados

Vamos descomplicar os termos técnicos que você vai encontrar:

Firewall

📖 Definição simples: Um segurança na porta da sua rede que decide quem entra e quem sai.

💡 Analogia: Como o porteiro de um prédio que só deixa entrar visitantes autorizados.

IP (Internet Protocol)

📖 Definição simples: O "endereço" do seu computador na internet. Como o número da sua casa.

💡 Exemplo: 192.168.1.10 - cada número é único como um CPF digital.

Porta (Network Port)

📖 Definição simples: Como uma "porta" virtual que programas usam para se comunicar.

💡 Exemplo: Porta 80 = sites comuns, Porta 443 = sites seguros (HTTPS), Porta 25 = emails.

Modelo OSI

📖 Definição simples: Uma "receita" de 7 passos para a comunicação entre computadores.

💡 Analogia: Como uma carta passa por: escrever → colocar no envelope → endereçar → enviar → transportar → entregar → abrir.

Stateful

📖 Definição simples: Firewall que "lembra" das conversas que estão acontecendo.

💡 Analogia: Como um porteiro que anota quem entrou e deixa essas pessoas saírem e voltarem sem problema.

TCP vs UDP

📖 Definição simples: Dois jeitos de enviar dados.

💡 TCP: Como uma conversa com confirmação ("Recebeu?" "Recebi!"). UDP: Como gritar na rua ("Fulano, te espero lá!" - sem esperar resposta).

DMZ (Zona Desmilitarizada)

📖 Definição simples: Uma área "neutra" entre sua rede interna e a internet.

💡 Analogia: Como a recepção de um prédio. Visitantes podem chegar até lá, mas não entram nos escritórios internos sem autorização.

IPS (Prevenção de Intrusão)

📖 Definição simples: Um sistema que detecta e BLOQUEIA ataques em tempo real.

💡 Analogia: Um detector de metais no aeroporto que não só avisa, mas também impede a entrada.

❓ Perguntas Frequentes (FAQ) - Respostas Simples

🔒 Resposta: São coisas diferentes! O antivírus é como um médico que examina arquivos que JÁ ESTÃO no seu computador. O firewall é como um segurança que controla quem ENTRA e QUEM SAI. Um não substitui o outro - eles trabalham juntos!

🎯 Analogia: Antivírus = detector de metais no aeroporto. Firewall = controle de quem entra e sai do país.

🌐 Resposta: Não! O firewall usa uma política de "negar tudo, permitir explicitamente". Isso significa que ele bloqueia tudo QUE NÃO FOI AUTORIZADO. Coisas que você usa no dia a dia (navegar em sites, assistir vídeos, mandar emails) são permitidas porque usam portas padrão como 80, 443, etc.

💡 Dica: É como um prédio que só deixa entrar moradores e entregadores autorizados - você não quer que qualquer estranho entre, mas quer receber suas encomendas!

🚪 Resposta: Portas são como "canais" que programas usam para se comunicar. Algumas portas são padrão: 80 (sites), 443 (sites seguros), 25 (email). Portas perigosas são aquelas que programas maliciosos podem abrir para "escutar" comandos de hackers.

⚠️ Exemplo: Se um malware abre a porta 4444, um hacker pode se conectar nessa porta e controlar seu computador. O firewall bloqueia portas desconhecidas!

🏠 Resposta: O firewall do roteador protege sua CASA INTEIRA (todos os dispositivos). Mas se um dispositivo já dentro de casa for infectado, o firewall do roteador não vai proteger os outros. Por isso, ter firewalls nos próprios computadores (host-based) é uma camada extra de segurança!

🎯 Analogia: Firewall do roteador = portão da rua. Firewall do computador = porta de cada casa. Ideal ter os dois!

🎯 Em Resumo: O que você precisa saber sobre Firewalls

Firewall = segurança que controla quem entra e sai

Quanto mais "inteligente", mais camadas ele inspeciona

Stateless = olha só endereço (mais rápido, menos seguro)

Stateful = lembra de conversas (equilíbrio)

Proxy = revista conteúdo (mais seguro, mais lento)

NGFW = inteligente + detecta ameaças (recomendado)

Lembre-se: Na segurança, não existe "mágica". O firewall é uma ferramenta poderosa, mas faz parte de um sistema maior. Combine com antivírus, mantenha tudo atualizado e, principalmente, tenha consciência - o elo mais fraco da segurança ainda é o ser humano!
