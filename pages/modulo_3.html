3.0.1 Por que devo fazer este módulo?

Protocolos são a base das comunicações de dados. Por esta razão, eles têm sido alvo de agentes ameaçadores há muito tempo.

Os analistas de segurança cibernética devem entender como os atores de ameaças usam recursos de protocolos comuns em ataques cibernéticos.

3.1.1 IPv4 e IPv6

O IP foi projetado como um protocolo sem conexão de Camada 3. Ele fornece as funções necessárias para entregar um pacote de um host de origem a um host de destino por meio de um sistema interconectado de redes. O protocolo não foi projetado para rastrear e gerenciar o fluxo de pacotes. Essas funções, se necessárias, são executadas principalmente pelo TCP na camada 4.

Vulnerabilidade Crítica:

O IP não faz nenhum esforço para validar se o endereço IP de origem contido em um pacote realmente veio dessa origem.

Impacto:

Agentes de ameaças podem enviar pacotes usando endereço IP de origem falsificado (spoofing) e adulterar outros campos do cabeçalho IP para realizar seus ataques.

PARA ANALISTAS DE SEGURANÇA:

É importante que os analistas de segurança entendam os diferentes campos dos cabeçalhos IPv4 e IPv6 para identificar ataques como spoofing, fragmentação maliciosa e exfiltração de dados.

Contém um valor binário de 4 bits definido como 0100 que identifica isso como um pacote IPv4.

🔍 Caso de Uso: Atacantes podem enviar pacotes com versão inválida (ex: 0111) para causar erros de processamento em firewalls e sistemas de detecção mal configurados, fazendo com que o pacote seja ignorado ou processado incorretamente.

Um campo de 4 bits contendo o comprimento do cabeçalho IP. O comprimento mínimo de um cabeçalho IP é de 20 bytes.

🔍 Caso de Uso: Ataque "Ping of Death" explorava valores inconsistentes entre IHL e tamanho total, causando buffer overflow em sistemas antigos ao forçar a alocação de memória incorreta.

Anteriormente chamado de campo Tipo de serviço (ToS), o campo DS é um campo de 8 bits usado para determinar a prioridade de cada pacote.

Os seis bits mais significativos do campo DiffServ são o Ponto de código de serviços diferenciados (DSCP).

Os dois últimos bits são os bits de notificação de congestionamento explícito (ECN).

🔍 Caso de Uso: Ataques de QoS podem manipular campos DSCP para priorizar tráfego malicioso ou causar negação de serviço em redes que confiam nesses valores para alocação de banda.

Especifica o comprimento do pacote IP, incluindo o cabeçalho IP e os dados do usuário.

O campo de comprimento total é de 2 bytes, portanto, o tamanho máximo de um pacote IP é de 65.535 bytes, no entanto, os pacotes são muito menores na prática.

🔍 Caso de Uso: Ataques de amplificação (como Smurf) usam pacotes com comprimento total manipulado para causar sobrecarga na rede da vítima.

À medida que um pacote IP se move pela Internet, talvez seja necessário cruzar uma rota que não possa lidar com o tamanho do pacote. O pacote será dividido, ou fragmentado, em pacotes menores e remontado posteriormente.

Esses campos são usados para fragmentar e remontar pacotes.

💣 Caso de Uso Real: Ataque Teardrop (1997) - Explorava fragmentação maliciosa com offsets sobrepostos. Atacantes enviavam pacotes fragmentados onde o segundo fragmento tinha offset menor que o tamanho do primeiro, causando crash no sistema ao tentar remontar. Sistemas Windows 95 e NT 4.0 eram vulneráveis.

Contém um valor binário de 8 bits que é usado para limitar a vida útil de um pacote. O remetente do pacote define o valor inicial do TTL e este é subtraído de um toda vez que o pacote é processado por um roteador.

Se o campo TTL for decrementado até zero, o roteador descartará o pacote e enviará uma mensagem ICMP de tempo excedido para o endereço IP de origem.

🕵️ Caso de Uso: Traceroute usa TTL para mapear rota - envia pacotes com TTL=1,2,3... e coleta mensagens ICMP de tempo excedido. Ataques de "traceroute" podem ser usados para mapeamento de rede.

O campo é usado para identificar o protocolo de próximo nível. O valor binário de 8 bits indica o tipo de carga de dados que o pacote está carregando, o que permite que a camada de rede transfira os dados para o protocolo apropriado das camadas superiores.

Valores comuns incluem: ICMP (1), TCP (6) e UDP (17).

🔍 Caso de Uso: Ataques de protocolo anomalia - enviar pacotes TCP na porta 80 com protocolo ICMP (1) pode confundir firewalls que esperam TCP (6) para tráfego web.

Um valor calculado com base no conteúdo do cabeçalho IP. Usado para determinar se algum erro foi introduzido durante a transmissão.

🔍 Caso de Uso: Ataques de corrupção de checksum podem ser usados para esconder alterações maliciosas no cabeçalho, fazendo com que o pacote seja descartado por roteadores legítimos mas processado por sistemas de inspeção.

Contém um valor binário de 32 bits que representa o endereço IPv4 de origem do pacote. O endereço de origem IPv4 é sempre um endereço unicast.

⚠️ Principal vulnerabilidade: IP Spoofing - falsificação do endereço de origem. Usado em ataques DDoS de amplificação (DNS Amplification, NTP Amplification) onde o atacante envia pequenas requisições com IP da vítima como origem, e servidores respondem com grandes volumes de dados para a vítima.

Contém um valor binário de 32 bits que representa o endereço IPv4 de destino do pacote.

🔍 Caso de Uso: Ataques de broadcast - enviar pacotes para endereços de broadcast (ex: 192.168.1.255) pode causar tempestades de resposta na rede local.

Este é um campo que varia em comprimento de 0 a um múltiplo de 32 bits. Se os valores de opção não forem um múltiplo de 32 bits, 0s serão adicionados ou preenchidos para garantir que este campo contenha um múltiplo de 32 bits.

💣 Caso de Uso: Ataque de Opções IP (1990s) - Roteadores antigos processavam opções IP em software (não em hardware), causando sobrecarga de CPU. Atacantes enviavam pacotes com múltiplas opções para degradar performance do roteador.

Um pacote IPv6 também pode conter cabeçalhos de extensão (EH) que fornecem informações opcionais da camada de rede. Opcionais, os cabeçalhos de extensão ficam posicionados entre o cabeçalho IPv6 e a carga. EHs são usados para fragmentação, segurança, suporte à mobilidade e muito mais.

Ao contrário de IPv4, os roteadores não fragmentam os pacotes IPv6 roteados.

Este campo contém um valor binário de 4 bits definido como 0110 que o identifica como um pacote IPv6.

Este campo de 8 bits é equivalente ao campo IPv4 Differentiated Services (DS).

Este campo de 20 bits sugere que todos os pacotes com o mesmo rótulo de fluxo recebem o mesmo tipo de tratamento pelos roteadores.

🔍 Caso de Uso: Ataques podem manipular rótulos de fluxo para tentar bypassar políticas de QoS ou para fingerprinting de implementações IPv6.

Este campo de 16 bits indica o comprimento da porção de dados ou carga útil do pacote IPv6.

Este campo de 8 bits é equivalente ao campo do protocolo IPv4. Ele exibe o tipo de carga de dados que o pacote está carregando, permitindo que a camada de rede transfira os dados para o protocolo apropriado das camadas superiores.

Valores comuns: TCP (6), UDP (17), ICMPv6 (58).

Este campo de 8 bits substitui o campo TTL IPv4. Esse valor é subtraído de um por cada roteador que encaminha o pacote.

Quando o contador chega a 0, o pacote é descartado e uma mensagem ICMPv6 de Tempo Excedido é encaminhada ao host emissor, indicando que o pacote não atingiu seu destino por causa do limite de saltos.

Este campo de 128 bits identifica o endereço IPv6 do host de envio.

⚠️ Vulnerabilidade: Apesar de mais seguro, IPv6 também é vulnerável a spoofing. Ataques de amplificação usando IPv6 são possíveis, embora mais complexos devido ao espaço de endereçamento maior.

Este campo de 128 bits identifica o endereço IPv6 do host receptor.
